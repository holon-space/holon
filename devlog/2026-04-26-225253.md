## Investigating and Fixing `gpui_ui_pbt` Test Stalls

**Context**: The property-based test for GPUI was stuck rendering "Loading…" placeholders across all three panels, preventing any block interaction. Root cause analysis revealed three compounding issues: an overly permissive readiness gate, a UI dispatch bypass that hides failures, and a cascading live_query watcher failure.

**What happened**:
- Traced window-readiness logic in `gpui_ui_pbt.rs:208-211` — the `has_content` check counts `loading` builder widgets (wrapped via `tag()` as `widget_type: "loading"`) as "content", so the window greenlights before panels populate
- Discovered `ClickBlock(LeftSidebar)` short-circuits through `sut.rs:910-922` via `synthetic_dispatch` → `engine.dispatch_intent_sync`, bypassing GPUI's mouse pipeline entirely — test claims success without touching UI
- Identified that `live_query` inner render in `builders/live_query.rs:14-29` falls back to loading slot when `LiveQueryView` entity isn't in `ctx.local.live_queries` — the structural transition from `loading()` builder to real `column(...)` never populates this map
- Confirmed that `render_entity.rs` has three early-exit paths (cached entity, missing block_id, focused-block) that skip the `tracked()` wrapper, meaning `render-entity-{id}` isn't always registered

**Challenges**:
- **Live_query cascade**: Panel chrome renders but inner `live_query` stays on loading slot — `walk_for_entities` discovery likely runs once and never re-runs after structural transition
- **Entity lookup inconsistency**: Sidebar rows wrap as `selectable(...)` directly (no outer `render_entity()`), so registered element is `selectable-{row_id}` not `render-entity-{block:id}` — a simple lookup strategy would panic
- **Timing race**: `services.dispatch_intent` from click handler is non-awaitable while `apply_intent` goes through `dispatch_intent_sync` — immediate `Edit*` after `ClickBlock` can race

**Decisions**:
- Implement fallback entity lookup chain: try `render-entity-{id}` → `selectable-{id}` → generic `find_by_entity_id` — handles all three render_entity early-exit cases
- Add `wait_for_focus_to_match(region, block_id)` barrier after `ClickBlock` to close race window before downstream edit actions
- Tighten readiness gate to wait for actual block content in Main panel (first `render_entity-…` or real block widget), not just any non-`live_block` element

**Status**: Ready to implement Phase 1 (entity lookup chain + focus barrier + readiness gate refinement) with verification checks for entity histogram dumps and focus-match waits.
