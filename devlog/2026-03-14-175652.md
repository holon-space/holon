## Added Configurable Glass Background Effect to GPUI Frontend

The GPUI frontend lacked a way to enable macOS's frosted glass window effect shown in design mockups. We added a user-configurable setting to toggle window-level backdrop blur.

**What happened:**

- Added `glass_background: bool` field to `UiSettings` in `instance_config.rs`, defaulting to `false`
- Wired the setting into the preferences dispatch system via `set_preference` for `"ui.glass_background"`
- Updated window creation in `lib.rs` to read the setting and apply `WindowBackgroundAppearance::Blurred` when enabled
- Reduced main content background opacity to 70% when glass mode is active, allowing the desktop blur to show through semi-transparent panels
- Added `PreferenceDef` in `define_preferences()` to expose "Glass Background" as a toggle in the Appearance section of the settings UI

**Challenges:**

- `PrefSection` needed to be `Clone` for the preference definition — was already implemented, just required verifying the trait bound
- Window background appearance must be set at window creation time, so the feature requires a restart to take effect

**Decisions:**

- Made glass mode semi-transparent (70% opacity) rather than fully opaque, so the blurred content behind the window is actually visible
- Kept it opt-in (defaults to `false`) since it has performance implications on some systems

**Status:** Glass background toggle now appears in Settings → Appearance and correctly blurs the desktop wallpaper through semi-transparent UI panels on macOS (requires restart).
