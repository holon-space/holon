## PBT-Driven Bug Fixes: Block Tree Chord Operations

Fixed six bugs in block tree chord operations (Indent, SplitBlock, MoveUp) by driving fixes through Property-Based Testing reproducer, systematically unmasking deeper layers with each fix.

### What happened

- **Bug X (traits.rs:779)**: `split_block` assigned new block's old content to `"title"` property instead of `content` field—left new block empty
- **Bug Y (sut.rs:2195)**: SUT doubled `block:` prefix when mapping split IDs (`EntityUri::block` called on already-prefixed string)
- **Bug Z (sut.rs:2655)**: org-file comparison didn't resolve synthetic split IDs to real UUIDs before comparing on-disk state
- **Bug A (parser.rs)**: Parser assigned default `sort_key = "a0"` to all siblings post-parse; added `assign_per_parent_sort_keys` post-pass to assign unique fractional indices per parent group
- **Bug D (block_params.rs)**: `build_block_params` omitted `sort_key` from SQL params—OrgSyncController UPSERTs always landed at SQL default `"a0"`
- **Bug E (assertions.rs)**: Reference state comparison included `sort_key` (implementation detail); normalized it away in `normalize_block`

### Challenges

- **Layered failure modes**: Each fix exposed the next bug; required 5 PBT iterations to surface all issues
- **Fractional index semantics**: Sort keys are production-only optimization for `get_prev_sibling`; reference state uses sequence order—had to decouple comparison logic
- **Synthetic ID mapping**: Split operations create new UUIDs in backend but reference state needed mapping before org-file diff to avoid false divergence

### Decisions

- **PBT-first approach**: Reproduced each bug through minimal failing test before fixing, preventing false fixes
- **Deferred Bug C**: Complex chord-op chain divergence (Indent→SplitBlock→MoveUp→Indent) involves reference state's move semantics; documented for next session rather than introduce risky changes

**Status**: Five bugs fixed; PBT progresses 11 transitions before hitting original Bug C (prev-sibling order tracking under chained ops). Build clean, all tests passing.
