## Cleaned up build artifacts from version control

The repository had accumulated large Flutter and Gradle build artifacts (`.dart_tool/`, `build/`, `.gradle/`) that were already tracked in version control, preventing `.gitignore` patterns from taking effect.

**What happened:**

- Ran `jj st` and discovered 27+ files exceeding the 1.0 MiB snapshot limit (trace logs up to 9.9 MiB, SQLite WAL files, heap dumps, Gradle caches)
- Identified root cause: `.gitignore` patterns don't prevent commits of *already-tracked* files; they only block untracked files
- These build artifacts were likely committed in early initialization commits (e.g., `flutter_rust_bridge_codegen`)
- Global `~/.gitignore` contains patterns for `**/flutter/build/`, `**/.dart_tool/`, etc., but jj respects the git index for already-tracked files
- Used `jj file untrack` command to remove tracking while preserving files on disk
- Files can now be properly ignored by `.gitignore` going forward

**Decisions:**

- Used `jj file untrack` instead of `git rm --cached` to align with jj workflow
- Kept `.gitignore` patterns in place rather than increasing `snapshot.max-new-file-size` to prevent similar bloat in future

**Status:** Build artifacts removed from version control; `jj st` now respects ignore patterns for Flutter and Gradle caches.
