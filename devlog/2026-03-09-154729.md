## Refactoring Frontends to Use Typed ViewModel with Macro-Generated Dispatch

**Context**: The frontends were duplicating arg extraction logic from `RenderExpr`, losing type safety. The goal is to have each frontend's builders consume pre-destructured typed parameters from `ViewModel`/`NodeKind`, with the `builder_registry!` macro auto-generating the dispatch match from function signatures.

**What happened**:

- **Extended `NodeKind` enum** with missing fields that frontends were extracting: `Text` now has `size` and `color`, `Icon` has `size`, `Spacer` has `width`/`height`, container variants have `gap`, `SourceBlock` has `editable`, `PieMenu` has `fields`
- **Enhanced `builder_registry!` macro** with `node_dispatch` mode that reads `pub fn render(...)` signatures from source files to extract parameter names
- **Macro generates destructuring matches** automatically — e.g., `NodeKind::Badge { label, .. } => badge::render(label, ctx)` — so builders are simple typed functions with no NodeKind knowledge
- **Special case for full-node access**: if first param is named `node`, the macro passes the whole `&ViewModel` instead of destructuring
- **Updated shadow builders** (in progress) to populate all the new `NodeKind` fields from `ResolvedArgs`
- **Fixed pattern matches** across `view_model.rs` to handle new struct variants and use `..` for forward compatibility

**Challenges**:

- **Field exhaustiveness**: Many pattern matches needed `..` to ignore new fields; scattered throughout the file in `pretty_print()`, tests, and helper functions
- **Spacer variant change**: Changed from unit `Spacer` to struct `Spacer { width, height }`, requiring updates to all match arms and constructors
- **Macro signature parsing**: Had to implement parsing of `pub fn render(...)` to extract typed parameters and map them to `NodeKind` variant fields

**Status**: `NodeKind` extended with all missing fields, pattern matches fixed, macro enhanced with signature parsing — ready to rewrite shadow builders to populate the new fields and then migrate frontend builders to use typed parameters.
