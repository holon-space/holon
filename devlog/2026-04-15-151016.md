## Refined Plan: Extract Org Format Layer from `holon-orgmode`

**Context:** The current plan conflates "hygiene refactor" with "enable wasm parsing" and makes false claims about WASM viability without auditing actual code. This version separates concerns, fixes validated bugs, adds real verification, and names targets honestly.

**What happened:**
- Audited all moved modules for `std` disk calls and found `parser.rs:36-37` calls `std::fs::canonicalize` — must be refactored before move
- Discovered `models.rs` re-exports from `holon-filesystem` (pulling `walkdir` + `tokio[fs]`), defeating "purity"
- Found `dioxus-web` is workspace-excluded with zero org imports; real WASM consumer is `holon-worker` on `wasm32-wasip1-threads`, not `wasm32-unknown-unknown`
- Confirmed workspace `uuid` lacks `"js"` feature and `chrono` lacks `wasmbind` — both will fail on browser wasm
- Validated that `build_block_params` is gated under `di` feature inside `org_sync_controller.rs`, making it not-pure
- Identified 15 files consuming these modules; any using module-path imports (e.g. `holon_orgmode::parser::X`) will break on move

**Challenges:**
- **Impurity in "pure" code**: `generate_file_id` does filesystem canonicalization. Solution: extract to separate trait method, make it optional via builder or feature.
- **Models tangled with filesystem**: Re-exporting `Directory` from `holon-filesystem` is a category error. Solution: move `Directory` back into `holon-orgmode` or change re-export to be explicit/feature-gated.
- **Feature unification risk**: `uuid` and `chrono` workspace defaults won't compile on `unknown-unknown` wasm. Solution: explicit `default-features = false` posture + separate audit if worker ever targets browser.

**Decisions:**
- Rename `holon-org-serde` → `holon-org-format` (avoid confusion with the serde crate)
- Split is for **hygiene + Markdown reuse**, not yet for WASM; drop false motivation claims
- Move only parsing/rendering/diff logic; leave `build_block_params` in `holon-orgmode` until SQL shim is extracted separately

**Status:** Plan rewritten with explicit bug fixes (impurity), correct target identification (`wasip1-threads` if ever wasm'd), backward-compat audit tasks, and verification that doesn't assume facts not in evidence. Ready for implementation phase.
