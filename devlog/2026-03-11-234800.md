## Implement Entity Profile Preservation Through Render Pipeline

**Context:** The GPUI main panel was rendering blocks as flat text instead of structured widgets with icons, spacers, and editable fields. Root cause: the entity profile information (e.g., `row(icon("orgmode"), spacer(8), editable_text(...))`) was being stripped when converting `ResolvedRow` to `HashMap<String, Value>` at multiple points in the pipeline.

**What happened:**

- **Core type ergonomics**: Added `impl Default for ResolvedRow` and `ResolvedRow::plain(data)` constructor to `widget_spec.rs` to support convenient construction.
- **Render evaluation**: Updated `sorted_rows()`, `OutlineTree`, and `partition_screen_columns()` in `render_eval.rs` to carry `Vec<ResolvedRow>` through the pipeline instead of stripping to plain `HashMap`.
- **RenderContext refactor**: Changed `data_rows: Vec<ResolvedRow>` (from `Vec<HashMap>`), added `profile()` accessor, updated `with_row`/`with_data_rows` signatures and `BlockRenderCache::get_or_watch()` to preserve profiles instead of stripping.
- **Render interpreter fixes**: Updated `shared_render_block_build()` to check `ba.ctx.profile()` and return `ProfileWidget(expr)` for non-source blocks with profiles; added `RenderBlockResult::ProfileWidget` variant.
- **Shadow builder updates**: `render_block.rs` interprets `ProfileWidget` expressions; `columns.rs` and `pref_field.rs` extract `.data` only when needed; widget builder macro fixed to pass `__row.data` to `ViewModel::element()`.
- **Frontend entry point cleanup**: All 8 frontends (GPUI, Blinc, Ply, TUI, Dioxus, WaterUI, MCP, Flutter) now pass `widget_spec.data.clone()` directly instead of stripping profiles; added `ProfileWidget` handling in render_block builders.
- **Navigation update**: `TreeNavigator::from_rows()` updated to work with `&[ResolvedRow]`.

**Status:** Implementation complete and verified. All crates compile (`holon-api`, `holon-frontend`, `holon-macros`, and 8 frontends). All existing tests pass (41 in holon-frontend, holon-api tests green). Ready for end-to-end verification with GPUI describing blocks via MCP.
