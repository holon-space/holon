## Enhanced PBT Navigation Testing with Region CDC Verification

The PBT was verifying navigation cursor correctness but not validating that displayed blocks actually change after navigation. The region data infrastructure (`region_data`, CDC streams) was completely absent from the test.

**What happened:**

- Discovered that invariant check 8 (region data verification) was a no-op — `region_data` was never populated because `region_streams` were never set up
- Identified that the test only verified `current_focus` materialized view, not the blocks that should be displayed post-navigation
- Traced the navigation system: `NavigateFocus` changes `focus_roots`, which should trigger CDC events that update region subscriptions
- Confirmed the root layout query uses `from children` (context-dependent) and relies on CDC to reflect navigation changes in `region_data`
- Planned enhancement: wire up `query_and_watch` infrastructure to populate region CDC streams, then enhance invariant checks to verify displayed blocks match navigation state

**Decisions:**

- Use the existing `query_and_watch` API to subscribe to region queries and capture CDC events, mirroring production app behavior
- Track expected `region_data` in the reference model state — when `NavigateFocus` occurs, update expected children based on the focused block's actual children
- Add validation that post-navigation `region_data` matches reference model, ensuring displayed content reflects the navigated-to context

**Status:** Architecture designed and implementation ready to begin — will add region stream setup to `StartApp`, enhance reference state tracking, and expand invariant checks to verify block visibility after navigation.
