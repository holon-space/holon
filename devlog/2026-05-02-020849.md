**Critical bugs caught by v5 review:**

- **Storage layout permutation**: `[sort..., rowid, non_sort..., weight]` physical order doesn't match logical column order. Reading via `column(i)` returns wrong values; `HashableRow` hashes mismatch between btree and DBSP overlay, breaking weight cancellation on in-tx UPDATEs
- **`IndexInfo.has_rowid` must be `false`**: For index pages, `rowid()` reads `record.last_value()` which is `weight`. Setting `has_rowid: true` causes silent weight leakage
- **REFRESH MATERIALIZED VIEW path missing**: `translate/view.rs:471-544` opens cursors via `CursorType::BTreeTable`, which would corrupt a leaf-index page
- **`IndexInfo::new_from_index` loses NULLS order**: Must build `IndexInfo` field-by-field, not via constructor
- **`BTreeCursor::new_with_index_info` doesn't exist**: Must be added
