## Applied Rust build optimizations from Turso workspace to holon

The holon workspace needed the same build speed improvements that were previously applied to the Turso codebase. We ported profile optimizations, parallel codegen settings, and unified feature compilation via `cargo-hakari`.

**What happened:**

- Added dev profile optimizations to `Cargo.toml`: `split-debuginfo = "unpacked"` (faster macOS linking), `debug = "line-tables-only"` (minimal debug info), and `[profile.dev.build-override] opt-level = 3` (optimized proc macros/build scripts)
- Configured release profile with `codegen-units = 4`, `panic = "abort"`, and `lto = "thin"`
- Added `-Z threads=8` rustflag in `.cargo/config.toml` for Apple targets to enable parallel LLVM codegen (requires nightly)
- Created `rust-toolchain.toml` pinning the project to nightly
- Initialized `cargo-hakari` workspace-hack across all 18 workspace crates to unify feature sets and prevent recompilation of shared dependencies (serde, tokio, futures, etc.) with different feature combinations
- Fixed nightly compatibility issue in turso fork by adding `#![recursion_limit = "256"]` to `turso_core/lib.rs`, pushed to holon branch, and updated holon's dependency

**Challenges:**

- `-Z threads=8` requires nightly toolchain; switched from stable after confirming nightly was acceptable
- Build lock contention from other concurrent cargo processes initially blocked verification
- `turso_core` had pre-existing recursion depth issues on nightly; resolved by modifying the fork and updating holon's git dependency

**Status:** Build succeeds with all optimizations active. Incremental builds should be noticeably faster on subsequent compilations.
