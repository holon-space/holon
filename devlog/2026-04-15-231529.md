## Comprehensive Code Review and Plan Refinement for Container Queries

### Context
A plan to implement responsive layout via space-aware interpretation was submitted. The plan claimed to fix `if_space` inside positional `columns` children, but a senior code review revealed the central mechanism doesn't work: positional children are pre-interpreted with the parent's full `available_space` before `columns` ever runs, making any downstream `set_space` calls on static results a no-op.

### What happened
- **Identified fatal flaw (H1)**: Positional children in `columns` are interpreted with parent context at widget-builder time (`widget_builder.rs:280-285`), so `if_space` branches are decided before partitioning happens. Plan's `set_space` plumbing cannot retroactively fix this for static results.
- **Verified secondary issues (H2â€“H6)**: Resize doesn't propagate (stale captured space), `set_space` silently fails on non-Reactive kinds (violates "fail loud"), `GpuiRenderContext` isn't safely Clone, GPUI render-time narrowing is dead code, and zero automated test coverage for the critical shape.
- **Discovered actual use case was different**: User clarified the real need is root/any-level **viewport profile breakpoints** (Case A â€” already shipping, zero code needed) *and* **per-slot container queries inside `columns`** (Case B â€” requires the fix).
- **Verified Case A works**: `entity_profile.rs:31-46` already whitelists `available_width_px` in Rhai, `set_viewport` bumps `ui_generation` on resize, and `pick_active_variant` re-evaluates profiles against fresh `available_space` on every interpret. Users can author responsive profiles in YAML today.
- **Rewrote plan for Case B**: `columns` must become a `raw fn` builder (like `if_space`) that lazy-interprets positional children against a partitioned `RenderContext` *before* they see it, not after. Dropped dead-code GPUI render plumbing, added automated test (EntityProfile with viewport `when` on a block_ref inside a slot), verified profile resolution reads the narrowed context correctly so no matcher changes needed.

### Challenges
- **Pre-interpreted children bypass partitioning**: Root cause was architectural â€” the widget builder's static collection path resolves all positional exprs eagerly. Fix requires switching `columns` from typed extraction to `raw fn` + lazy interpretation.
- **Silent failures**: `set_space` on `Static`/`StaticCollection` returns silently. Added `debug_assert!` demand to fail loud if called on unsupported kinds.
- **No test surface**: Manual Rhai tests were planned; replaced with automated test in the test suite exercising the real shape (EntityProfile conditions reading narrowed space).

### Decisions
- **Case A delegated**: Root/any-level viewport profiles need zero Rust code; tracked as authoring task (YAML profiles), not code task.
- **Case B uses lazy interpretation**: Rather than threading mutables through static view trees, `columns` will interpret children with pre-narrowed `RenderContext`. Aligns with existing `if_space` pattern and avoids signal plumbing into static leaves.
- **Reuse profile infra**: No profile-matcher changes needed; the narrowed context flows naturally through `pick_active_variant` since it already re-evaluates conditions against `ctx.available_
