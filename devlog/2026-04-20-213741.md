## Override pathfinder_simd with SIMD rename fix fork

The project was pinned to a specific nightly toolchain due to breaking changes in the upstream `pathfinder_simd` crate. A fork with fixes needed to be integrated to allow using the latest nightly.

**What happened:**
- Located `pathfinder_simd` v0.5.5 as a transitive dependency from crates.io
- Added `[patch.crates-io]` override in `Cargo.toml` pointing to `https://github.com/pbdeuchler/pathfinder` on the `pbd/fix_simd_renames` branch
- Discovered the crate lives in the `simd/` subdirectory of the workspace repo, so configured the patch with the appropriate path
- Ran `cargo update` to resolve lock file to commit `1562d7bb` from the patched branch
- Verified the override took effect by checking `Cargo.lock`
- Unpinned the nightly toolchain in `rust-toolchain.toml` from `2026-04-19` to track latest nightly

**Status:** `pathfinder_simd` now resolves to the fork with SIMD renames fixed; project is able to build with latest nightly toolchain.
