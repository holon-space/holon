## Debugging Empty Sidebar and Missing Doc-Block Names in holon-live

The holon-live instance was showing an empty left sidebar with no doc-block names, despite data existing in the database. Logs revealed repeated `flat_driver diff received, target_len=0` messages and a render cascade loop (~550 EditorView renders per 5s).

**What happened:**
- Verified 325 blocks in DB with a valid root document structure (`block:bb859b3e-...`)
- Confirmed sidebar query (`from block | filter name != ""`) returns correct 4 rows when executed directly
- Discovered `describe_ui` reports the sidebar as a `table [30 items]` instead of `list [4 items]` — filters and render expressions not applied
- Found critical parsing error in logs: **`clickable()` function not registered** in the render DSL parser
- Traced render cascade: structural signal fires every 2-3s, triggering new `ReactiveView` instances with empty `MutableVec`, but each restart aborts before data arrives
- Root layout's 3 block_refs all rendering as `(empty)` — likely same cascading restart issue

**Challenges:**
- The render DSL parser falls back silently to `table()` when `clickable()` parsing fails, masking the real error
- Reactive view restarts (`start()` calls) fire faster than the data driver can deliver initial diffs, leaving views perpetually empty

**Status:** Root causes identified — missing `clickable()` DSL function and render cascade restart loop preventing data delivery. Ready to implement fixes in reactive_view startup logic and render DSL registration.
