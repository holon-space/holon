## Loro Subtree Sharing: Backend Operations & Iroh Integration

We needed to enable sharing Loro sub-trees across peers using Iroh sync — specifically, a way to advertise a subtree on one side and mount it on the other, gated behind the `iroh-sync` feature flag.

- Resolved `#[affects_entity]` macro question (doesn't exist; used `#[affects("parent_id")]` instead for both share ops)
- Implemented `Ticket` module (`sync/ticket.rs`): versioned, base64-wrapped format with strict decode validation
- Created deterministic peer-id derivation (`sync/share_peer_id.rs`) from device secret + shared tree ID for CRDT hygiene
- Built `iroh_advertiser.rs`: persistent accepter pool with background lifecycle management (`start_share` / `drop_share`)
- Extracted `sync_doc_handle_connection` handshake logic so the advertiser could reuse it without duplication
- Implemented `SubtreeShareOperations` trait via `#[operations_trait]` on `LoroShareBackend`, registered as `OperationProvider` on entity `"shared_tree"`
- Added comprehensive edge-case unit tests (bad ticket, unknown block, unknown retention) and threat model docs

**Challenges:**
- `#[affects_entity]` didn't exist — tracked down that only `#[affects("field")]` is available; used parent block ID as the affected field
- `get_typed` not in scope during adapter refactor — inlined the block lookup instead of creating a helper
- Pre-existing Iroh networking test flakes (mdns/relay unavailable locally) — marked network-dependent test as `#[ignore]`

**Decisions:**
- Deterministic peer-id from secret key (not `rand::random`) — ensures consistent identity across restarts for the CRDT
- Separate `LoroShareBackend` entity instead of adding methods to `LoroBackend` — cleaner separation, avoids feature-gate pollution
- DI wiring gated on `iroh-sync` feature — Loro share operations only available when explicitly enabled

**Status:** All backend operations complete and tested (53 sync tests passing). UI wiring deferred to Phase 2. Ready for frontends to call `execute_operation("shared_tree", "share_subtree", {...})` and exchange tickets.
