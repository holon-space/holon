## TUI Frontend Migration to New Architecture

**Context**: The TUI frontend (`@frontends/tui/`) was outdated and excluded from the workspace, relying on a removed `holon-prql-render` crate. It needed to be updated to match the modern architecture used by the GPUI frontend.

**What happened:**

- **Dependencies updated**: Removed `holon-prql-render`, `holon`, `libsql`, `ferrous-di`; added `holon-frontend` and `holon-macros`. Re-enabled TUI in workspace `members`.
- **State management rewritten**: Replaced old `BackendEngine` + raw CDC `RowChange` stream with `FrontendSession` + `AppState` (Arc<RwLock> wrapper) + `watch_ui()` UiEvent stream, matching GPUI's pattern.
- **Rendering pipeline refactored**: Replaced custom `RenderInterpreter` with shadow builder pattern using `builder_registry!` macro. Created 14 TUI-specific builders (text, row, list, checkbox, badge, icon, spacer, editable_text, section, block, columns, tree, block_ref, render_entity, live_query, error).
- **Main entry point modernized**: `main.rs` and `app_main.rs` now use `FrontendSession::new()`, `watch_ui()` for CDC, and `RenderInterpreter<TuiWidget>` for paint ops.
- **Old code cleaned up**: Removed `ui_element.rs`, `render_interpreter.rs`, `launcher.rs`, old PBT files, and obsolete test infrastructure.

**Challenges:**

- **API evolution in r3bl**: The `Surface` trait's layout API had changed. Resolved by importing the correct `LayoutManagement` trait from the local r3bl fork.
- **Builder selection**: Initially implemented all builders as passthroughs. Simplified to only TUI-compatible widgets; the variant selection system filters out GUI-only ones.
- **r3bl trait bounds**: `TuiState` needed `Clone + Debug` for r3bl's handler trait. Used `Arc<RwLock<_>>` wrapper on `AppState` to satisfy this.

**Decisions:**

- **Shared builder reuse**: Leveraged `holon-frontend`'s shared builders (`shared_tree_build`, `shared_block_ref_build`, etc.) rather than reimplementing in TUI.
- **TuiWidget as intermediate type**: Simplified render ops pipeline by using a small `TuiWidget` enum as intermediate representation before flattening to r3bl render ops, rather than complex direct conversion.

**Status**: TUI builds cleanly with no errors. Architecture fully migrated; ready for testing against real UI definitions.
