## Devlog: Dynamic Entity Schema Registration via DI and MCP Support

**Context:**
The newly refactored `GraphSchemaRegistry` hardcodes entity registration in `registration.rs`. To support external data sources (MCP servers, Todoist, filesystem) and user-defined schemas from YAML, we need a scalable, trait-based registration system that collects entity definitions at runtime.

**What happened:**
- Reviewed `EntityConfig` (YAML runtime schema) and `EntitySchema` (struct compile-time schema) type mismatch — both needed unified handling
- Identified 5 related improvements: (1) `EntitySchemaProvider` trait for DI collection, (2) Todoist/Filesystem GQL integration, (3) SQL type converters, (4) DDL unification via struct annotations, (5) unused import cleanup
- Discovered Block/Document tables have extra columns (`depth`, `sort_key`, `collapsed`, `block_type`) not on the structs — these are computed/denormalized, not entity state
- Designed implementation order: static providers first (Todoist, Filesystem), then SQL type converters, DDL schema annotations, import cleanup, then MCP dynamic registration last

**Challenges:**
- **Compile-time vs runtime schemas** — `EntitySchema` derives from `#[entity]` macros, `EntityConfig` from YAML. Solution: add `EntityConfig::to_entity_schema()` method for runtime construction, same shape
- **Extra DDL columns** — Block/Document tables have application logic columns (depth, sort_key) not in entity structs. Solution: add optional `#[schema(default = "...")]` annotations for computed columns, keep entity structs clean
- **Type mapping** — SQL types (`VARCHAR`, `INTEGER`) ↔ Rust types need bidirectional conversion. Solution: implement `Into<SqlType>` on `FieldType` enum, use in both `EntityConfig::to_schema()` and new struct codegen

**Decisions:**
- Trait `EntitySchemaProvider` collected via `ferrous-di::get_all_trait()` — reuses DI pattern already used for `OperationProvider`
- Block/Document DDL stays hand-written SQL in `CoreSchemaModule` during #4, but is now validated against struct schema via compile-time test
- MCP entity registration via `McpSchemaModule` implementing `EntitySchemaProvider`, built from loaded `EntityConfig`s at DI init time

**Status:** Plan finalized; implementation will proceed in order: (1) `EntitySchemaProvider` trait + DI collection, (2) Todoist provider, (3) SQL type converters, (4) DDL schema validation, (5) cleanup, then (2-dynamic) MCP runtime registration.
