## Fixed Materialized View Timeout and Updated Stale Document Schema References

The Left Sidebar was failing to render due to a 30-second materialized view creation timeout caused by a schema mismatch.

**What happened:**

- Traced the "Failed to create materialized view" error by analyzing SQL execution logs with `HOLON_TRACE_SQL=1`
- Found that `execute_ddl_with_deps` was waiting indefinitely (30s timeout) for a `document` resource that never became available
- Discovered the root cause: the Left Sidebar's PRQL query in both `assets/default/index.org` and `holon-pkm/index.org` referenced `from document`, but that table was removed in a recent refactor — documents are now `block` rows with `is_document = 1`
- Updated the PRQL query in both files from:
  ```prql
  from document
  filter name != "" && name != "index" && name != "__default__"
  ```
  to:
  ```prql
  from block
  filter is_document == 1
  filter name != "" && name != "index" && name != "__default__"
  ```
- Restarted the app and confirmed all four render_entity calls now succeed (root-layout, left sidebar, main panel, right sidebar)

**Challenges:**

- The error message was vague; required tracing through the dependency resolution system (`matview_manager.ensure_view()` → `execute_ddl_with_deps()`) to find the actual issue
- The SQL trace logs were at DEBUG level but the filter was set to INFO, masking the actual DDL being attempted — had to infer the problem from timing (exactly 30s timeout) and resource names
- Needed to cross-reference schema changes to understand that `document` table no longer exists

**Status:** Left Sidebar now renders successfully; materialized view timeout resolved.
