## Debugged and Fixed PBT Test Failures in E2E Integration Tests

Enhanced the `general_e2e_pbt.rs` tests with ViewModel de-compilation, which exposed several bugs. Ran the tests and systematically analyzed each failure.

**What happened:**

- Identified `render_entity()` fails when `block_ref()` templates encounter text blocks (blocks without query sources) — production bug in `block_domain.rs`
- Found region watches only initialized for `Region::Main`, causing failures when PBT navigates to `LeftSidebar`/`RightSidebar` — test setup bug
- Discovered CDC drain timeout (20ms) insufficient for IVM propagation after mutations, especially through chained materialized views
- Added diagnostic logging to understand CDC event flow and matview state
- Implemented `render_leaf_block()` fallback for non-root text blocks and set up watches for all regions
- Increased drain timeouts to 200ms and added `tokio::task::yield_now()` to allow scheduler to process CDC tasks
- Added `refresh_watches()` fallback that re-queries active watches directly when CDC doesn't arrive in time

**Challenges:**

- **CDC timing cascade**: Region navigation goes through `navigation_cursor → focus_roots → region join`, with IVM propagation delays exceeding initial 20ms timeout. Root cause: each matview hop adds latency, requiring 200ms+ for reliable convergence
- **Turso IVM behavior unclear**: Some UPDATE operations may not trigger CDC events on base tables, necessitating the re-query fallback pattern
- **Region watch coverage gap**: Only `Main` region initialized, but PBT generates all three regions—missed in test setup

**Status:** `general_e2e_pbt` (Full variant) now passes. `sql_only` variant has remaining pre-existing bug in org parser's custom TODO keyword extraction unrelated to these changes.
