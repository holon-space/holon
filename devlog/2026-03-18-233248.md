## Fixed Missing Cursor in Main Panel EditableText

The main panel content became non-editable because the editor reconciliation wasn't finding `EditableText` nodes in the rendered view.

**What happened**

- Diagnosed via logs and source inspection that `reconcile_editors()` only walked the root ViewModel tree
- Discovered that `LiveQuery.content` in the root tree is always interpreted with empty data rows (a skeleton structure)
- Found that actual content lives in `LiveQueryView::current` and `BlockRefView::current` (populated asynchronously via watch channels)
- Modified `reconcile_editors()` to also walk ViewModels from `BlockRefView` and `LiveQueryView` entities
- Added `current_view_model()` accessor methods to both `BlockRefView` and `LiveQueryView`
- Tested and confirmed editing now works

**Challenges**

- Initial investigation required tracing through multiple async watch channels and shadow builders to understand the ViewModel tree structure
- The root ViewModel appeared to have content, but it was only a skeleton with empty data rows
- Realized the async views store their actual ViewModels separately and needed direct access to them

**Decisions**

- Extended `reconcile_editors()` to walk nested ViewModels rather than creating a separate reconciliation pass, keeping the logic centralized in the render phase where all entities are available

**Status**

Editing in the main panel is working again — cursors appear and content is fully editable.
