## EntityUri Refactoring: Eliminating String-Based URIs in Integration Tests

**Context:** The codebase had scattered `&str` parameters for document and block URIs where `EntityUri` should be used instead. This extended from integration tests into production code, requiring a cross-crate refactoring to ensure type safety.

**What happened:**
- Started in `assertions.rs`: changed `block_belongs_to_document(doc_uri: &str)` to accept `&EntityUri`, updated `find_document_for_block()` to return `Option<EntityUri>`
- Updated data structures: `reference_state.rs` changed `documents: HashMap<String, Document>` → `HashMap<EntityUri, Document>` and `block_documents: HashMap<EntityUri, EntityUri>`
- Pushed EntityUri into production traits: `BlockReader::iter_documents_with_blocks()` return type changed from `Vec<(String, Vec<Block>)>` to `Vec<(EntityUri, Vec<Block>)>`
- Updated production code across `di.rs`, `org_sync_controller.rs`, `org_renderer.rs`, and document operations
- Fixed integration test files: `state_machine.rs`, `sut.rs`, `generators.rs`, `phased.rs`, `test_environment.rs`
- Resolved cascading errors in `holon-frontend`: `watch_ui()`, `spawn_watcher()`, `render_context.rs` to accept owned `EntityUri` for `tokio::spawn` compatibility
- Fixed parser call sites: `parse_org_file()` now takes `&EntityUri` for parent directory, updated callers to use `EntityUri::doc_root()`

**Challenges:**
- **Scope creep into production code:** Initially intended for integration tests only, but discovered String URIs throughout the trait layer and controllers. Pushed through the full dependency chain rather than converting at boundaries.
- **Ownership and async:** `tokio::spawn` requires `'static` values. Changed `watch_ui()` and `run_reactive_watcher()` from taking `&EntityUri` references to owned `EntityUri`, with strategic cloning for spawned futures.
- **Agent drift:** Mid-session agents converted EntityUri back to String at call sites (opposite direction). Reverted and proceeded manually with careful attention to pushing EntityUri deeper, never converting outward.

**Status:** Compilation passes across holon, holon-orgmode, holon-integration-tests, and holon-frontend. Created follow-up task to remove `.org` file extensions from document URIs in the reference model.
