## Dismantling Global UI View Registries in `FocusRegistry`

The previous refactor to remove global registries missed `FocusRegistry.editor_inputs` and `FocusRegistry.editor_views` in `entity_view_registry.rs`. These fields duplicate existing parent-owned mechanisms and leak `Entity` instances indefinitely due to missing unregister calls.

**What happened:**

- Analyzed both registry fields: `editor_views` (render-pass deduplication) and `editor_inputs` (four scattered roles: focused row lookup, forward entity lookup, cursor read, and MCP debug introspection)
- Discovered the infrastructure to replace them already exists: `LocalEntityScope::get_or_create` for editors, `UiState.focused_block` for focus tracking, and `engine.watch_editor_cursor()` for cursor state
- Designed a 6-step removal plan, ordered from low-risk (MCP debug cleanup) to high-risk (move cursor subscription into `EditorView`)
- Started **step 1**: Move Enter/MoveUp/MoveDown capture handlers from page-level `lib.rs` into `EditorView::render`, threading `nav`, `focus`, and `focused_eid` through 9 `GpuiRenderContext::new` call sites across `lib.rs`, `reactive_shell.rs`, `views.rs`, and editor code
- Updated 5 files mid-refactor; identified 5 remaining call sites needed to restore compilation

**Challenges:**

- The handlers were captured at page scope but needed row context — solved by moving them into `EditorView::render` where both scope and row ID are available
- Threading new parameters through deeply nested call chains required updating both `GpuiRenderContext` and intermediary types (`ReactiveShell`, `LiveQueryView`, `RenderEntityView`) — documented each call site for clarity

**Decisions:**

- Ordered removals to make each step independently shippable and unblock the next
- Chose to move cursor subscription into `EditorView` rather than maintain a central `pending_cursor` field, so editors own their state

**Status:** Working copy is mid-refactor and won't compile yet. Detailed handoff written to `HANDOFF_FOCUS_REGISTRY.md` with design analysis, removal plan, current state of 5 modified files, and the 5 remaining call sites blocking step 1 completion.
