## DI Framework Migration Conflict Resolution

A feature branch using the deprecated `ferrous_di` DI framework was rebased onto main, which had migrated to `fluxdi`. This created 10 directly conflicted files plus dozens of auto-merged incompatibilities.

### What happened

- Ran `jj resolve --tool mergiraf` on the conflicted commit — resolved 2 of 10 files automatically
- Read all 8 remaining conflict files with `snapshot` markers to understand the DI framework split
- Determined the destination (main/fluxdi) versions were the correct base; took those for all conflicted files
- Discovered ~40 additional auto-merged files that referenced old `ferrous_di` APIs — systematically reverted these to destination versions
- Removed rebased-revision additions: `core_module.rs`, `event_infra_module.rs`, `bootstrap.rs` module references, and related mod.rs declarations
- Fixed callers expecting old API signatures (`create_test_engine` tuple returns, module re-exports)
- Reverted `ply/` builder files that the rebased revision deleted but existed in destination
- Ran `cargo check` iteratively to surface remaining incompatibilities and fixed them

### Challenges

- **Framework-level mismatch**: The rebased revision's entire DI pattern (ferrous_di) was incompatible with destination's fluxdi — required understanding which files were core conflicts vs. auto-merged side effects
- **Cascading API breakage**: Fixing one file exposed 5–10 more files with incompatible callers (test helpers, re-exports, frontend modules) — used `cargo check` output to drive systematic fixes rather than trying to resolve files in isolation
- **Pre-existing issues**: Flutter crate (`rust_lib_holon`) had compilation errors in the destination too (`frb_generated` module missing) — confirmed this was unrelated to the rebase

### Decisions

- **Take destination wholesale for DI files** — Rather than merge ferrous_di and fluxdi patterns, took the fluxdi (destination) versions entirely since DI framework migrations don't compose
- **Revert over merge** — For the ~40 auto-merged files, reverting to destination was faster and safer than attempting surgical merges of incompatible APIs

**Status**: All conflicts resolved. Workspace compiles cleanly (`cargo check` passes for all crates except Flutter, which has pre-existing issues unrelated to the rebase).
