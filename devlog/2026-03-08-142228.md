## Implementing Shared Theming, Settings & Collapsible Sidebars

**Context**: The Holon frontends (Blinc, GPUI, Flutter, Ply) each implement theming, settings, and sidebar behavior independently. This plan centralizes abstractions in `crates/holon-frontend/` while keeping framework-specific code in each frontend, starting with Blinc and GPUI.

**What happened**:

- **Phase 2 (Settings)**: Extended `InstanceConfig` with `UiSettings` struct (theme name + widget state map). Added `WidgetState` with `open`, `width`, `height` fields. Implemented persistence via `InstanceConfig::save()` serializing back to TOML. Wired into `FrontendSession` with convenience methods: `ui_settings()`, `update_ui_settings()`, `widget_state()`, `set_widget_open()`. Internal state stored as `Mutex<(UiSettings, PathBuf)>` for thread-safe mutations.

- **Phase 3 (Collapsible Widgets)**: Added `widget_states: Arc<HashMap<String, WidgetState>>` to `RenderContext`, propagated through `clone_base()`. Extended `ScreenLayoutPartition` with `CollapsibleRegion<W>` struct containing `block_id: Option<String>` and widget. Updated `partition_screen_columns()` to extract block IDs from row metadata and return `CollapsibleRegion`. Updated Blinc and GPUI columns builders to look up widget state by block ID instead of hardcoded `sidebar_open` flag. Fixed all downstream consumers (Ply, shadow_builders).

- **Phase 1 (Theming)**: Created `crates/holon-frontend/src/theme.rs` with `Rgba8 = [u8; 4]` type and `ThemeColors` struct holding 14 RGBA color fields. Implemented direct conversions via `impl` blocks: `to_blinc_color()`, `to_blinc_color_tokens()`, `to_gpui_rgba()` behind feature flags (`blinc`, `gpui`). Added `ThemeDef` wrapper and `ThemeRegistry` for loading builtin YAML themes (embedded via `include_str!`) plus user themes from `~/.config/holon/themes/`. Custom YAML deserializer converts hex strings (`#RRGGBB`/`#RRGGBBAA`) to `Rgba8`. Wired into Blinc: created `HolonTheme` struct implementing `blinc_theme::Theme`, load registry at startup, pass theme colors to `ThemeState::init()`. Wired into GPUI: load theme registry, pass `ThemeColors` to `HolonApp`, replace hardcoded `rgb()` calls with theme lookups during rendering.
