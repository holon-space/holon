## PBT enhanced to catch stale text in split/join operations

The GPUI property-based test wasn't detecting that split and join operations left the UI displaying stale text while the DB and org file were correct. The issue was that `ElementInfo` only tracked structural properties (widget type, entity ID, bounds) but never compared rendered text against the reference state.

- **Diagnosed the root cause**: queried the running app's DB via MCP and confirmed DB and org file both showed the truncated/joined text correctly, but the on-screen `editable_text` widget still displayed the pre-operation string
- **Audited PBT coverage**: confirmed split operations do hit mid-text positions, but the test only validates block content in the DB (`assert_blocks_equivalent`), never on screen
- **Designed invariant without global registry**: extended `ElementInfo` with `displayed_text: Option<String>` (pushed into test-only `BoundsRegistry` during render) instead of adding a new global registry
- **Implemented the fix**:
  - Added `displayed_text` field to `ElementInfo` (`crates/holon-frontend/src/geometry.rs`)
  - Widened `tracked()` signature in `BoundsTracker`/`TransparentTracker` to accept the new field
  - Updated `editable_text` builder to snapshot live `InputState::value()` at render time and pass it through
  - Added `inv-displayed-text` invariant in `check_invariants_async` that asserts each on-screen block's text matches ref_state (skipping focused blocks to avoid timing issues)
- **Build status**: `holon-frontend` and `holon-integration-tests` compile cleanly; `holon-gpui` has pre-existing constructor-arg drift unrelated to this work that blocks running the full PBT

**Status**: Invariant wired and compiled; ready to run PBT once `holon-gpui` constructor signatures are updated. Full handoff doc written to `HANDOFF_DISPLAYED_TEXT_INVARIANT.md`.
