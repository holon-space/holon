## Rigorous Code Review & Plan Refinement: Persistent Shared Subtrees

**Context:** A plan to persist shared subtree snapshots across restarts was submitted. Before implementation, a senior code review uncovered three critical blockers, five medium-severity races, and several false assumptions about the existing codebase.

**What happened:**

- Read the proposed plan and traced through the actual implementation in `loro_module.rs`, `loro_share_backend.rs`, `loro_document_store.rs`, and `iroh_advertiser.rs` using parallel Explore agents
- Discovered `device_key` is **regenerated every process startup** (not persisted), breaking the entire `stable_peer_id` strategy across restarts
- Found that `save_to_file` performs non-atomic overwrites, and corrupt snapshots are **auto-deleted** on load — a data-loss hazard when the shared doc is the only copy
- Identified ordering race: fork-prune happens in-memory, advertiser broadcasts before disk persistence completes, leaving orphaned snapshots if process crashes mid-sequence
- Validated 10 key hypotheses against the codebase; 8 were falsified (existing code does not debounce saves, does not persist device keys, does not auto-cleanup killed tasks, etc.)
- Rewrote the plan to add five mitigations: persist `device_key` to disk, atomic snapshot writes with corruption recovery, enforced save ordering, separate event channel for degradation signals, and PBT-first verification strategy

**Challenges:**

- **Blocker #1 (device_key):** Fresh generation breaks peer identity stability. Fixed by adding ~20 LOC to persist/load `SecretKey` to/from `{storage_dir}/device.key`.
- **Blocker #2 (atomicity + auto-delete):** Non-atomic writes + corruption-triggered deletion = silent data loss. Fixed by atomic write (tmp+fsync+rename) and corruption handling that renames file to `.corrupt-<ts>` and emits `ShareDegraded` signal instead of deleting.
- **Blocker #3 (ordering race):** Global doc save happens asynchronously *after* advertiser broadcasts. Fixed by enforcing: save shared snapshot → save global doc → register → advertise → return.

**Decisions:**

- **PBT-first verification:** Rather than writing integration tests, extended `subtree_share_round_trip_pbt` in `sync_pbt.rs` with action streams (`EditOnA/B`, `RestartA/B`, `CorruptSharedOnA`, `SettleSaves`, `PullBtoA`) and nine invariants covering all blockers. Avoids flaky timing-dependent tests.
- **Separate degradation channel:** Use new `DegradedSignalBus` instead of retrofitting into closed `EventKind` enum, avoiding coupling to Block event system.
- **Debounce reduction + shutdown flush:** Changed from speculative 500 ms to 150 ms, added explicit `flush_all()` on graceful shutdown, and swapped `JoinHandle` drop for `AbortHandle` or `CancellationToken`.
