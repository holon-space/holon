## Android V1 Validation Spike: Manifest IME Resize Path

**Context:** Testing whether Android's `windowSoftInputMode="adjustResize"` is sufficient to fold the IME into the safe-area rect, or whether we need to fall back to the GameActivity + JNI callback path per the plan.

**What happened:**
- Applied `windowSoftInputMode="adjustResize"` to `AndroidManifest.xml:16`
- Fixed two cross-compile blockers: `tokio-tungstenite` native-tls → `rustls-tls-webpki-roots`; explicit `RUST_FONTCONFIG_DLOPEN=on` cargo env
- Rebuilt NDK artifact (aarch64-linux-android release)
- Relaunched emulator with `-gpu host` to use Metal backend instead of SwiftShader
- Confirmed `gpui_mobile::android::window::update_safe_area_from_content_rect` instrumentation is live — observed baseline `safe_area_insets updated: top=0 bottom=0 left=0 right=0` in logcat
- Attempted GL and Vulkan GPU paths on the emulator; both failed due to driver bugs before reaching interactive state

**Challenges:**
- **Vulkan path:** emulator's `vulkan.ranchu.so` returns unmappable `vk::Result`s → `DeviceError::Unexpected` → device lost 37ms after adapter selection. With debug extensions, the driver SEGVs in `vkSetDebugUtilsObjectNameEXT`.
- **GL path:** adapter limits insufficient for gpui's `instances_layout` bind-group; wgpu-core reports "underlying API does not support enough features to be a fully compliant implementation."
- Root cause: Android emulator on macOS M4 has fundamental incompatibilities with gpui's GPU abstraction layer (both Vulkan and GL translation layers are broken). Real Android device is the path of least resistance.

**Decisions:**
- Did not commit emulator-specific workarounds (GL-only, webgl2 downlevel limits) to the holon zed fork — these regress real devices and are emulator-only hacks
- Pivoted to rebasing holon's zed fork onto latest upstream, which may pick up genuine improvements

**Status:** V1 manifest flip confirmed wired and instrumentation live, but IME delta cannot be empirically measured on this emulator. Recommend real Android device for final validation. Side effect: rustls swap for tokio-tungstenite is a genuine cross-compile fix worth upstreaming. Hard gate (peaceful-strolling-muffin §8 streaming row/col) still unshipped.
