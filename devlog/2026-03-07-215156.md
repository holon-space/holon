## Wiring up region watch for navigation-dependent block verification in PBT

The PBT was verifying navigation state (`current_focus` cursor) but never checking that the correct blocks were actually displayed—invariant 8 was dead code because `region_data` and `region_streams` were never populated.

### What happened

- **Added `expected_focus_root_ids(region)` to `ReferenceState`** — mirrors the SQL `focus_roots` matview logic: returns the focused block + its children (blocks with matching `parent_id`)
- **Added `setup_region_watch(region)` to `TestEnvironment`** — creates a chained matview query joining `focus_roots` with `block` via `query_and_watch`, storing initial data and CDC stream for region verification
- **Wired region watch into StartApp transition** — calls `setup_region_watch(Region::Main)` after engine init so the chained matview exists before navigation tests run
- **Rewrote invariant 8** — now iterates over all regions and verifies displayed block IDs match `expected_focus_root_ids()` from the reference model
- **Fixed reference model bugs** — discovered and fixed two pre-existing issues:
  - **Seeded default blocks**: reference model wasn't rewriting `document_id` from `doc:index.org` to `doc:__default__` (line 856-857)
  - **Source block children**: reference model assigned them `document_id: sentinel:no_parent` instead of inheriting from parent (line 756)

### Challenges

- **Chained matview propagation**: The region watch creates a `focus_roots JOIN block` matview that depends on the `focus_roots` matview created by navigation logic. When navigation changes `focus_roots` via IVM, CDC must propagate through the chained join—this is a known Turso IVM edge case
- **Reference model initialization order**: Seeded blocks get inserted and then rewritten, but the rewrite at line 856-857 only patched `parent_id`, missing `document_id`

### Status

Code compiles and implements the full region watch chain. Pre-existing `document_id` mismatches in reference model fixed. Test ready to verify navigation-dependent block display through CDC events.
