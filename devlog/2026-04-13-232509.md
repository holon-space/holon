## Implementing User Driver Fail-Loud Geometry and Quiescence Helpers

The user driver trait had accumulated silent fallbacks that masked missing bounds and timing issues during test execution. This session refined the trait contract and implemented fail-loud semantics for better test debuggability.

**What happened**

- Audited all user driver implementations (`GpuiUserDriver`, `ReactiveEngineDriver`, `EnigoUserDriver`, `FlutterUserDriver`) to identify silent degradation paths
- Reclassified call sites into three categories: canonical post-input dispatch (Cat A), synthetic debt (Cat B), and intentional test-only bypasses (Cat C)
- Adapted to a parallel refactor that renamed `apply_ui_mutation` → `synthetic_dispatch` (removing deprecation entirely)
- Implemented fail-loud behavior in `GpuiUserDriver::click_entity` and `scroll_at`: now returns `Err` instead of silently degrading when element bounds aren't available
- Added `wait_for_element_bounds` helper to `ReactiveEngineDriver` — waits for GPUI's `BoundsRegistry` to transition bounds from `staged` to `committed` state (mandatory before first interaction on newly-created elements)
- Removed bounds-missing silent fallback from `send_key_chord` while preserving the legitimate `extra_params` fallback for split-block dispatch
- Removed the trait-internal `#[allow(deprecated)]` and cleaned up stale doc references

**Challenges**

- **Parallel refactor collision:** Mid-session, another change renamed the deprecated method, breaking assumptions about compiler enforcement. Adapted by treating the rename as cosmetic and focusing on geometric fail-loud + quiescence helpers as the real value.
- **BoundsRegistry staged/committed lifecycle:** Bounds live in `staged` until the next frame pump. Tests that create an element and interact within one reactive cycle see `None`. The wait helper is mandatory, not optional — documented with a guard in `click_entity`.
- **Enigo's layered silent paths:** `Enigo::new()` failure was a separate untouched fallback. Now fails loud alongside bounds-missing, making the CI contract explicit.

**Status**

Steps 1–4 (trait refactor, fail-loud geometry, quiescence helper, Enigo cleanup) are implemented. Pending: validation tasks (V1 MCP scroll audit, V2 bounds-miss baseline, V3 EnigoUserDriver liveness check) and documentation of the `render-block-{id}` prefix constraint on non-block entities.
