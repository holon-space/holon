## Implement WHERE Clause Pushdown for MCP Virtual Tables

We needed to enable dynamic URI parameter binding in MCP foreign data wrappers, allowing WHERE clauses to populate template parameters instead of requiring static configuration.

**What happened:**

- Added `FetchMode::ResourceTemplate` enum variant to store URI templates and default parameters separately from static resources
- Modified `build_fdw_metadata()` to auto-register key columns for empty `uri_params` entries, marking them as required and equality-constrained
- Updated `McpForeignDataWrapper::new()` to detect dynamic params (empty string values in YAML config) and use `ResourceTemplate` mode instead of `Resource`
- Implemented WHERE constraint handling in `McpCursor::filter()` to merge pushed-down predicates into template parameters, expand the URI, and fetch the result
- Added error handling for missing required parameters (when no WHERE clause provides the needed value)
- Added two new test cases validating the dynamic parameter detection and template expansion flow

**Decisions:**

- Config convention uses empty strings (`session_id: ""`) to mark dynamic parameters, and non-empty strings for static values — this is backward-compatible and explicit
- Template expansion happens at filter time rather than table init, enabling late binding of constraint values
- Missing required parameters return an error rather than silently proceeding, preventing ambiguous empty results

**Status:** All 6 tests pass. WHERE clause pushdown for MCP virtual tables is fully implemented in `crates/holon-mcp-client/src/mcp_vtable.rs`.
