## Implement Window Close to Exit Application

The GPUI frontend was continuing to run in the background after closing the window, leaving a orphaned process. A single-window app should terminate when that window is closed.

**What happened**

- Investigated GPUI's window lifecycle APIs to understand how to handle close events
- Reviewed Zed's implementation and GPUI storybook examples for patterns
- Located the window opening code in `frontends/gpui/src/lib.rs` inside a `cx.spawn` block
- Added `window.on_window_should_close()` callback immediately after `open_window()` that calls `cx.quit()`
- Verified the change compiles without errors
- The fix now properly terminates the application run loop when the close button is pressed

**Decisions**

- Used `on_window_should_close()` rather than trying to detect "last window removed" events, since Holon is a single-window application where any close equals app exit
- Registered the callback inline within the `open_window` closure to keep the logic colocated with window creation

**Status**

Window close now properly exits the application. The `holon-gpui` process terminates immediately when the window's close button is clicked, rather than orphaning a background process.
