## Fluent Emoji Icon System Setup

We needed a multi-color icon system that matches the warm, alive visual style from the vision doc, with the ability to grey out icons when they shouldn't distract and colorize them when needed.

**What happened:**

- Selected **Fluent Emoji (flat style)** from Microsoft as the best fit — multi-color SVGs with real-world colors, MIT licensed, warm professional aesthetic
- Created `assets/icons/` directory structure with `manifest.toml` for declarative icon listing
- Built `download.sh` script that fetches only needed SVGs from the official GitHub repo on demand, parsing the manifest and skipping already-present files
- Tested initial 32 icons (checkmark, notebook, fire, calendar, etc.) — all downloaded cleanly, ~42 KB total, individual files 0.5–2.6 KB each
- Added `.gitignore` to keep SVGs out of VCS (manifest + script stay checked in)
- Integrated into `justfile` with `just icons` commands and made `just build` depend on icon sync — fresh clones now work out-of-box

**Challenges:**

- Needed to avoid bloating the repo while keeping the system frictionless — solved by storing only the manifest, downloading on first build
- Required a robust parser for the manifest format to map local names to Fluent Emoji's naming convention — used TOML for simplicity and added error handling for missing files

**Decisions:**

- **On-demand download over VCS** — keeps repo lean, SVGs are large and don't need version control
- **Manifest-driven** — adding icons is as simple as one line in TOML, no manual file management
- **Integrated to build pipeline** — `just build` now handles icon setup transparently

**Status:** Icon system ready. 32 test icons downloaded and verified. To add icons, edit `assets/icons/manifest.toml` and run `just icons`. Available emoji names browsable at [fluentemoji.com](https://fluentemoji.com/).
