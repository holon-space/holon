## Critique & Refactor of Enchanted Booping Twilight Plan

**Context:** A senior code review of the MCP hub architecture uncovered 8 implicit assumptions, 2 confirmed bugs, and several missing implementation details that would have blocked progress.

**What happened:**

- Validated the plan against actual codebase (reactive.rs, rmcp v0.12, holon-worker Cargo.toml, serve.mjs)
- Found `snapshot_resolved()` doesn't exist on `ReactiveEngine` — correct method is `snapshot()` which already does BlockRef resolution
- Found wire protocol double-serializes `arguments` as `"{\"sql\":\"...\"}"` instead of plain JSON object — pure accidental complexity
- Identified missing `package.json` + `npm install ws` as explicit prerequisite
- Pinpointed WebSocket routing gap: `{ noServer: true }` + manual `server.on('upgrade')` needed to avoid catching all HTTP upgrades
- Discovered tick starvation bug: `poll_changes()` needs `block_on(sleep(Duration::ZERO))` to flush reactor before reading buffer
- Clarified single-port solution: serve.mjs proxies `/mcp/*` to relay on `RELAY_PORT=3002` using only Node.js built-ins
- Confirmed `frontends/mcp` is necessary (provides StreamableHTTP MCP protocol + tool schema validation)

**Challenges:**

- **Reconnect logic missing**: trunk `--watch` restarts serve.mjs on file save, killing WS connections permanently. Added explicit reconnect loops with backoff to both browser and native relay steps
- **Relay binary entrypoint**: `main.rs` always initializes `BackendEngine` from DB path, but relay mode has no local DB. Added relay-mode branch that skips engine init entirely
- **Vague dispatch table**: Plan listed `list_commands`/`execute_command` without specifying which trait to call on `HolonService`. Clarified this requires `ToolRouter::list_all()` from rmcp v0.12

**Decisions:**

- Use `ws` npm package + new `package.json` over manual HTTP Upgrade parsing (80+ lines, error-prone)
- Keep `frontends/mcp` for compile-time schema validation rather than reimplement StreamableHTTP in Node.js
- HTTP-proxy `/mcp` through serve.mjs to hide relay port complexity from Claude Code

**Status:** Plan refactored with all 8 assumptions made explicit, 2 bugs fixed, and relay infrastructure fully specified. Ready for implementation.
