## LogSeq-Inspired GPUI Frontend Styling & Configuration

**Context**: The GPUI frontend needed visual updates to match LogSeq's design language, and all hardcoded layout constants needed to be extracted into a configurable struct. Additionally, theme colors were not applying at runtime.

**What happened**:

- Launched a worktree agent to apply LogSeq-inspired design changes across 6 builder files: removed badge pill styling (plain teal text), enlarged section titles (16→28px bold), increased line height (22→26px), expanded tree item spacing (indent 24→28px, row height 28→32px), widened content padding (32→40px), and added CANCELLED state with red `✗` icon in `render_eval.rs`.
- Created `LayoutStyle` struct in `frontends/gpui/src/render/builders/style.rs` with all configurable size/spacing values and default implementations preserving current dimensions.
- Updated 11 builder files to reference `ctx.style()` instead of hardcoded constants, enabling future reactive configuration changes.
- Wired `LayoutStyle` into `GpuiRenderContext` with methods for retrieving and updating layout configuration.
- Fixed theme application by creating `apply_holon_theme()` function that maps holon's custom YAML `ThemeColors` (primary, text, background, sidebar, border, success, error, warning) onto gpui_component's global `ThemeColor`, called at window launch so theme changes take effect immediately.
- Verified both crates compile cleanly and launched the app for visual testing.

**Decisions**:

- Placed `LayoutStyle` as a `Mutable<T>` in render context to support future signal-based reactivity without rewiring the entire builder API.
- Kept theme color values unchanged from original YAML to preserve existing palette, only fixing the wiring so changes apply on startup.

**Status**: Both tasks complete — LogSeq styling applied, all sizes/spacing extracted into configurable `LayoutStyle` struct, and theme hot-reload fixed. App builds and runs successfully.
