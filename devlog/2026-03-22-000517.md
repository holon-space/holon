## Fixed multi-line text editor clipping in GPUI editable_text component

The editable text editor in `@frontends/gpui/src/render/builders/editable_text.rs` was collapsing to a single line, hiding any content after a line break. Text insertions would scroll out of view instead of expanding the editor.

**What happened:**
- Traced the rendering flow through `Input` component and `InputState` configuration
- Discovered `EditorView::render()` was using `.multi_line(true)` on the text input
- Analyzed `element.rs:request_layout()` to understand height calculation logic
- Found root cause: `multi_line(true)` sets height to `relative(1.0)` (100% of parent) with `min_size.height = line_height`, forcing single-line display
- Identified `auto_grow` mode as the correct API for dynamic height expansion
- Applied one-line fix: changed `.multi_line(true)` to `.auto_grow(1, usize::MAX)` in `editor_view.rs:44`
- Verified build succeeds with no new diagnostics

**Decisions:**
- Used `auto_grow` instead of `multi_line` because `multi_line` mode's height constraint resolves relative to parent dimensions, while `auto_grow` dynamically sizes based on actual content lines
- Set range `(1, usize::MAX)` to start at one row and grow without limit as text is added

**Status:** Fix applied and compiled successfully. Multi-line editors will now expand to show all content instead of clipping to single-line height.
