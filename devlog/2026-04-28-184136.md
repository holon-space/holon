## Decomposing Editor State Machine into Atomic Transitions

**Context:** The `split_block` bug (cursor position from in-memory edits, content from DB) exposes a deeper issue: the PBT bundles too many user actions into single transitions, missing combinatorial bugs. Rather than create specialized transitions, we should decompose the state machine into atomic editor primitives.

**What happened:**

- Identified that `SplitBlock`, `JoinBlock`, `EditViaViewModel`, and `TriggerSlashCommand` all bundle multiple independent user gestures (focus, type, cursor movement, structural mutations).
- Designed a new primitive set: `FocusEditableText`, `MoveCursor`, `TypeChars`, `DeleteBackward`, `PressKey`, `Blur` — each modifying only `InputState` until structural chords (Enter, Tab, Backspace-at-0) are pressed.
- Extended `ReferenceState` to track `ActiveEditor { block_id, in_memory_content, cursor_byte, popup_state }` — encoding the user's expectation that structural chords commit pending edits *before* mutating block tree.
- Ranked other complected transitions by bug-yield: `TriggerSlashCommand`/`TriggerDocLink` (popup leaks), `Indent`/`Outdent` (commit-before-mutate on any structural chord), `CycleStateToggle` (toggle with pending edits), `ClickToFocus` (focus-as-blur semantics).

**Challenges:**

- **State-space explosion:** Atomic primitives multiply reachable states. Mitigation: weight cheap ops (`MoveCursor`, single-char `TypeChars`) low; target 50-case runs initially.
- **Reference-model fidelity:** Must track cursor position and in-memory divergence without modeling IME, selections, or autocomplete. ASCII-only `TypeChars` keeps cursor byte-offsets exact per existing `send_key_chord` caveat.
- **Headless divergence:** The split bug requires real `InputState`. Headless mirror (shadow editor in SUT) is deferred; land in `gpui_ui_pbt` first.

**Decisions:**

- Keep `DragDropBlock`, `ConcurrentMutations`, `WriteOrgFile`, and sync primitives bundled — they are genuine single user gestures or external events.
- Encode "commit-on-structural-chord" as a core ref-model invariant: `PressKey(Enter)` on unfocused text must first call `set_field` with `in_memory_content`, then split. This catches production bugs automatically.

**Status:** Design complete and validated. Ready to implement step 1 (add `ActiveEditor` tracking to ref model in parallel with existing transitions to confirm green before adding new primitives).
