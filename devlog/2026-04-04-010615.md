## Architecture Documentation Audit & Cleanup

## Context
The project's architecture and documentation had drifted significantly from the current codebase state. A comprehensive audit was performed to identify what needed updating vs. removal.

## What happened

- **Audited ARCHITECTURE.md** against actual codebase and found 8 major discrepancy categories:
  - Crate structure missing `holon-engine`, `holon-integration-tests`, `holon-macros-test`
  - Frontends table severely outdated (4 listed vs. 8 actual: added GPUI/dioxus/ply/tui/waterui; Tauri marked as planned but doesn't exist)
  - `QueryableCache` signature changed from `<S, T>` generic to `<T>` only; trait bounds shifted from `HasSchema` to `IntoEntity + TryFromEntity`
  - `TursoBackend` replaced `ConnectionPool` with `DbCommand` channel + `DbHandle` actor pattern
  - `HasSchema` trait fully replaced by `IntoEntity`, `TryFromEntity`, `TypeDefinition`
  - CDC setup changed from `get_raw_connection()` callbacks to `cdc_broadcast.subscribe()`
  - Stale key files entries (deleted `holon-prql-render`, `core/transform/`, non-existent `external_system.rs`)
  - Standalone `holon-engine` Petri-net CLI not documented

- **Updated ARCHITECTURE.md** with all 10 corrections above; Entity Type System status changed from "Not Yet Implemented" to "Partially Implemented"

- **Audited docs/ directory** and classified 42 total files; found 7 completely obsolete files:
  - `tauri-mvp.md` — Tauri frontend never built
  - `implementation-summary.md` — References deleted `backend.rs`, `schema.rs`, `sqlite.rs`
  - `conversation.md` — Raw AI conversation from early design
  - `ARCHITECTURE_MULTI_VIEW_DECISIONS.md` — References deleted `RenderSpec`, `ViewSelector`
  - `HANDOFF_MULTI_VIEW_RENDER.md` — Describes deleted PRQL render-in-query syntax
  - `design/per-row-templates.md` — References deleted `holon-prql-render` crate
  - `LINEAGE_ANALYSIS_OPTIONS.md` — Stale lineage.rs references

- **Deleted** 7 obsolete docs + empty `docs/design/` directory; removed dangling Tauri reference from README.md

- **Left intact** 35 valuable docs (ADRs, handoffs, comparisons, tooling guides, historical reviews)

## Challenges

- **Distinguishing obsolete from historical**: Files in `codev/reviews/` reference deleted features but document valid past decisions — kept as historical record
- **README scope creep**: Full README rewrite needed (still describes project as "Rusty Knowledge - Iroh + Loro Integration") but deferred as separate task

## Decisions

- **Keep historical reviews** even when they reference deleted code — they document architectural evolution and past decisions
- **Update vs. delete trade-off**: ARCHITECTURE.md warranted full update (core reference doc); old design docs warranted deletion (superseded by newer ADRs)
