## Fixed panic in `render_block` when rendering empty template contexts

The GPUI frontend was crashing when rendering block templates with empty data rows, specifically when the parent context had no row ID.

**What happened:**
- Traced a panic at `entity_profile.rs:613` — `row_id(row)` called on empty `HashMap`
- Root cause: `shared_tree_build` (line 200-202) renders one template item using parent context when `data_rows` is empty
- If parent context also has an empty row (e.g., root render context), `render_block` receives `HashMap {}` with no `id` key
- The `_` branch at `render_interpreter.rs:405` then calls `resolve_row_profile(empty_row)`, which panics
- Added early return in `render_block`'s default branch: if row has no `id`, return `Empty` instead of attempting profile resolution
- Verified fix compiles and app runs stably (269MB RSS, no memory growth, clean shutdown)

**Status:** Panic fixed. GPUI frontend now handles empty template contexts gracefully without crashing.
