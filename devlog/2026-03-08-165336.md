## Extended `add-parent` alias to support optional target revision

The `add-parent` alias in `~/.config/jj/config.toml` only worked on the working copy (`@`), but needed to support rebasing a parent onto an arbitrary target revision.

**What happened**

- Identified that Jujutsu aliases don't natively support optional parameters — all args are positional and appended
- Refactored `add-parent` from a simple `rebase` command into a bash wrapper (using `util exec`) to handle optional arguments
- The alias now accepts `<new-parent>` as required arg and `<target-rev>` as optional second arg, defaulting to `@`
- Updated config with new implementation:
  ```toml
  add-parent = ["util", "exec", "--", "bash", "-c", """
  rev="${2:-@}"
  jj rebase -s "$rev" -o "parents($rev)" -o "$1"
  """, "--"]
  ```
- Tested with `jj add-parent r3bl-tui merge` — successfully rebased 7 commits to destination

**Decisions**

- Used bash subshell pattern (matching existing `rm-parent` alias) rather than creating a separate script file for consistency and portability
- Parameter expansion (`${2:-@}`) provides sensible default without requiring conditional logic

**Status**

Alias is working and tested. The command now supports both `jj add-parent <new-parent>` (targets `@`) and `jj add-parent <new-parent> <target-rev>` (targets arbitrary revision).
