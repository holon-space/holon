## Document Deduplication Fix and PBT Test Debugging

**Context**
The document creation deduplication logic was leaving documents in an inconsistent state. When `INSERT OR IGNORE` silently failed to insert a duplicate document, the `LiveDocumentManager` had no way to know the actual document ID, leading it to use wrong parent_ids in subsequent operations. Additionally, several pre-existing compile errors in the PBT test suite needed fixing.

**What happened**
- Fixed the `"create"` operation handler in `sql_operation_provider.rs` to SELECT the actual row after `INSERT OR IGNORE` and return its `id` in `OperationResult.response`
- Updated `LiveDocumentManager::create` in `di.rs` to use the returned document URI instead of querying LiveData (which has CDC race conditions)
- Discovered and fixed a reference model bug: `expected_focus_root_ids` was including the focus block itself, but the `focus_roots` matview only returns children; removed the incorrect inclusion
- Fixed multiple pre-existing compile errors in the PBT test code:
  - Removed redundant `wait_until_ready` calls (API no longer exists)
  - Updated `ReactiveQueryResults::snapshot()` calls to work with non-optional `RenderExpr`
  - Fixed closure signature mismatches in `ReactiveEngine` setup
- Verified the document dedup fix works: the "wrong parent_id" bug no longer appears in test output

**Challenges**
- **Chained matview investigation**: Initially suspected the region watch matview was suffering from Turso IVM chaining issues; added a direct `focus_roots` query to diagnose, but found the actual bug was in the reference model, not the matview
- **API drift**: The `ReactiveQueryResults` and rendering code had been refactored to use non-optional `RenderExpr`, but PBT code hadn't been fully updated; fixed all sites to match the new signatures

**Decisions**
- Return document URI in `OperationResult.response` rather than trying to reconstruct it later — gives the operation handler (which has DB access) responsibility for determining the truth
- Fixed reference model to match actual matview behavior rather than changing the matview — keeps the matview simple and avoids compounding Turso IVM issues

**Status**
Document dedup fix is working (original "wrong parent_id" bug fixed). PBT test compiles and runs further, now hitting a `BulkExternalAdd` timeout issue that appears to be a separate problem in how external file syncing resolves document URIs through the dedup map.
