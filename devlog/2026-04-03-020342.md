## Fixed logging filter in Holon GPUI

The TRACE and DEBUG logs from `turso_core` were appearing despite setting `turso_core=info` in `RUST_LOG` because the `EnvFilter` was configured incorrectly.

**What happened:**

- Identified that `.env` file propagation wasn't the issue — the root cause was in `logging.rs`
- The `EnvFilter` was being pushed as a standalone layer in a `Vec<Box<dyn Layer>>`, which doesn't actually filter events for other layers in the stack
- Refactored `logging.rs` to apply `EnvFilter` directly to each logging destination (file and stdout layers) using `.with_filter()`
- Each layer now respects the `RUST_LOG` environment variable independently

**Challenges:**

- Initially suspected `.env` file propagation with `dx serve` — added `.env` to `frontends/gpui/` but logs continued
- Discovered that `tracing_subscriber` layer composition requires the filter to be attached to each layer, not applied globally as a separate layer in the vec

**Decisions:**

- Applied `EnvFilter` per-layer rather than globally — this ensures consistent filtering across file and stdout outputs
- Kept the layered architecture intact while fixing the filter application

**Status:** Fixed logging filter configuration in `logging.rs`; turso_core TRACE/DEBUG logs should now be properly suppressed by `RUST_LOG=turso_core=info`.
