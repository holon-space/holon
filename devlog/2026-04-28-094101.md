## Shadow Board View: Lane Ordering, Empty Lanes, State Accent, and Row IDs

Continued work from the board-view handoff, implementing five shadow-layer follow-ups to unblock drag persistence and card rendering without waiting for the GPUI build to be fixed.

**What happened**

- Implemented stable **lane ordering** via `lane_order` prop on `board()`, with lexicographic fallback when absent
- Added **empty-lane handling** with `lane_label_default` argument (defaults to `"No status"`)
- Exposed `lane_field` as a top-level board property so GPUI render can read it later
- Wired `row_id` onto each card VM from the row's `id` column (groundwork for drag persistence)
- Created new `state_accent(state)` value-function that maps task state to color hex (sage/amber/coral/neutral)
- Registered `state_accent` in the widget-name DSL parser and wired it into `collection_profile.yaml::board_view`
- Extended board tests to cover both ordering paths, empty lanes, and end-to-end `state_accent` integration

**Decisions**

- Built all five features on the **shadow side only** to avoid the broken GPUI build; they're all either plumbing (lane_field, row_id) or pure value logic (state_accent) that don't require render changes
- Made `lane_order` default to `None` and only sort lexicographically when absent, rather than always sorting; allows explicit control from YAML profiles
- Kept `state_accent` simple (direct enum→hex mapping) rather than theme-aware, pending design feedback

**Status**

Five follow-ups shipped on shadow; all 7 tests pass including YAML profile reload. Drag persistence (#1) and richer card extraction (#4) remain gated on the GPUI signature refactor (unchanged blocker); shadow side is ready once that's unblocked.
