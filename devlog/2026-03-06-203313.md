## Fixed WatchHandle Channel Drop Bug in holon-gpui

The GPUI frontend was crashing with `ERROR holon::api::ui_watcher: [UiWatcher] Command channel closed unexpectedly for block 'block:root-layout'`, preventing structural CDC and re-renders.

### What happened

- Located the error at `ui_watcher.rs:72-79` where the command channel receiver got `None`
- Traced the root cause to `main.rs:200` — the `_command_tx` sender was created inside an async block but not included in the return value `Ok((session, app_state))`
- When the async block completed, Rust dropped `_command_tx`, closing the channel
- The `UiWatcher::run` loop detected the closed channel and logged the error, then exited
- This prevented any structural CDC or re-renders from working

### Root cause

In `main.rs`, the `watch_ui()` call returns both a `WatchHandle` (session) and a command channel sender:

```rust
let (session, command_tx) = watch_ui(...).await?;
// Only session was returned in Ok((session, app_state))
// command_tx was dropped here ↓
```

The `_command_tx` sender needs to stay alive for the lifetime of the application to keep the channel open.

### Solution

Modified `main.rs` to return the command sender from the async block and bind it in the outer `main()` scope as `_watcher_command_tx`. This keeps the channel alive for the entire application lifetime.

### Status

Error is fixed. The UiWatcher now initializes successfully with `watch_ui(block:root-layout) stream established` and begins rendering without channel closure errors.
