## Debugging GPUI Region Churn with Reproducible Seeds

**Context:** The `gpui_ui_pbt` test was failing at different steps with two related symptoms — CDC quiescence violations on `region:main` and focus propagation timeouts on Main panel clicks. Root cause investigation required deterministic reproduction.

**What happened:**

- Added `PROPTEST_SEED` environment variable support to `crates/holon-integration-tests/src/pbt/phased.rs` so test runs could be replayed
- Discovered that identical seeds produce *different* PBT transitions across runs due to async CDC arrival timing and GPUI render churn affecting precondition evaluation
- Scanned multiple seeds to find early-failing cases (seed=7 fails on pbt_step 13–14, seed=42 on step 12)
- Ran debugger session on seed=7 and confirmed `set_focus` *is* called from sidebar clicks via `selectable` builder → `dispatch_intent` → `reactive.rs:805`
- Identified a smoking-gun debug log pattern: `watch_view_e06c78ae5d0bb287` (the matview backing `region:main`) has **two subscribers**, causing duplicate CDC fanout

**Challenges:**

- LLDB lacks Rust expression evaluation, so stack frame inspection couldn't access local variables like `block_id`
- Precondition-based test state meant identical seeds didn't guarantee identical PBT paths; quiescence depends on wall-clock timing
- Test failure mode kept shifting between runs, making single-seed debugging less effective

**Key finding:**

The `region:main` CDC channel emits duplicate batches (`[Demux] subscribers=2`) on every focus change. The two subscribers are likely:
1. Test framework's `setup_region_watch(Region::Main)` 
2. Production `UiWatcher` for `block:default-main-panel`

This duplicate fanout causes spurious "add/remove cycles for unchanged data" that trigger `inv16` assertions and — downstream — cause the EditorView to be re-created mid-click, dropping focus updates.

**Status:** Root cause narrowed to duplicate subscriber fanout on `region:main` matview. Next: instrument `matview_manager.rs` subscribe path to confirm why two subscribers exist and whether lifecycle mismatch causes duplicate emissions.
