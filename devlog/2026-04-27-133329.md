## PBT Chord-Op Ordering Bugs: Complete Fix Pass

**Context**: The block-tree chord-op PBT was regressed after migration from synthetic dispatch to real input pipeline, surfacing multiple ordering failures in split, create, indent, and outdent operations that were hidden by a soft warning.

**What happened**

- Verified Bug B (Loro TreeID lookup) was already structurally fixed via `resolve_to_tree_id` in `loro_backend.rs:1372-1383`
- Promoted soft `eprintln!("WARNING:")` to hard `assert_eq!` in `assertions.rs:117` to surface hidden ordering bugs
- Fixed inv16 synthetic-id leak by resolving `block.id` through `doc_uri_map` before comparing to production draggables (`sut.rs:4707`)
- Fixed post-SplitBlock org-file ordering by changing renderer from `sequence`-based to `sort_key`-based ordering (`org_renderer.rs:57-61, 95-110`)
- Diagnosed and fixed root cause of Create/Indent/Outdent ordering divergence: `default_sort_key()` returned lower-case `'a0'` which lex-sorts **after** upper-case `FractionalIndex::to_string()` hex outputs (`'a'=0x61` > `'F'=0x46`), causing silent mis-ordering whenever a default-key block coexisted with generated keys
- Changed all `'a0'` defaults to `'A0'` across 8 files (block.rs, blocks.sql, frontend/lib.rs, todoist/models.rs, fixtures)
- Added ref-state Create to assign `sequence = max_sibling_sequence + 1` so canonicalization appends new blocks like production does
- Added `move_as_last_child` and `outdent_block` helpers to ref-state to mirror production's Indent/Outdent positioning semantics

**Challenges**

- **Lex vs fractional ordering**: `default_sort_key()` lower-case `'a0'` silently lex-sorts incorrectly against `FractionalIndex::to_string()`'s upper-case output. Fixed by normalizing to `'A0'` everywhere — a non-obvious pattern that would take hours to rediscover.
- **Soft warning masking hard bugs**: The original ordering assert was an `eprintln!` warning; promoting it to a hard panic revealed three additional distinct failures hiding behind it.

**Decisions**

- Ordered by `sort_key` (source of truth for fractional indexing) in renderer instead of stale parser-assigned `sequence` column
- Normalized all default sort keys to upper-case `'A0'` to match `FractionalIndex` case convention, preventing lex-comparison bugs

**Status**: All 7 chord-op ordering bugs (split, create, indent, outdent, synthetic-id leak) are closed. PBT now passes PROPTEST_CASES=20 with chord-op weights; next failure is an unrelated `ToggleState` precondition timeout in navigation layer (Bug X in kickoff doc).
