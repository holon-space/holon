## Profiling and Optimizing Recursive CTE Materialized Views

**Context**: Recursive CTEs in materialized views were showing severe performance degradation, scaling at O(n^1.85) instead of linear. Initial hypothesis was a nested-loop join in the DBSP circuit.

**What happened**:
- Built and ran comprehensive benchmarks across 50–25K row counts showing superlinear scaling (6.8s for 25K rows)
- Identified the nested-loop delta-delta join in `core/incremental/join_operator.rs:509-538` as suspected bottleneck
- Replaced it with a **hash join** using HashMap keyed by join key hashes (O(n) instead of O(n²))
- Discovered via profiling that the hash join change had no measurable impact — the real bottleneck was elsewhere
- Instrumented the full creation pipeline and discovered **99%+ of time was in the btree write phase** (`CommitState::UpdateView`)
- Further profiling revealed the actual culprit: **table data is re-read from scratch on every I/O yield from the circuit commit**

**Challenges**:
- Initial hypothesis (nested-loop join) was plausible but profiling showed the join operator only runs 1–4 iterations and takes <10ms total
- The circuit `commit()` method yields `IOResult::IO` for btree page allocations, which causes the VDBE to retry the entire CREATE MATERIALIZED VIEW instruction
- Each retry calls `reset_populate_state()`, which re-runs data collection (re-reading the full base table), but the circuit's internal `commit_state` machine persists and ignores the redundant input data
- This creates wasted re-reads: with ~14 I/O yields, the table is read 14 times unnecessarily

**Decisions**:
- Implemented hash join anyway as a latent performance improvement (removes O(n²) work even if not currently the bottleneck)
- Identified that the real fix requires preventing redundant table re-reads during I/O yields — either by making `populate_state` preserve circuit progress or by batching I/O yields differently

**Status**: Hash join optimization merged and tested (all 54 IVM tests pass). Root cause of slow creation is wasted table re-reads on circuit I/O yields — fix requires restructuring the state machine interaction between VDBE and recursive CTE circuit execution.
