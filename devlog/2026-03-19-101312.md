## Consolidated SQL Replay Tool with CLI

The `extract-sql-trace.py` script's functionality was being duplicated—log extraction logic existed separately from the SQL replay tooling. We consolidated both into a single Rust binary with a proper CLI.

**What happened:**

- Created new workspace member `tools/` with `turso-sql-replay` binary (single crate with `[[bin]]` section for future tools)
- Ported Python extraction logic from `extract-sql-trace.py` into Rust, using regex to parse `HOLON_TRACE_SQL=1` logs and inline parameter values
- Implemented 4 subcommands via `clap` derive macros: `extract` (parse logs → SQL file), `replay` (run with CDC/matview checks), `minimize` (6-phase crash minimizer), `run` (extract + replay in one shot)
- Merged existing replay and minimizer code from the old example
- Removed unused `try_replay` function and deleted `crates/holon/examples/turso_sql_replay.rs`
- Updated `SKILL.md` documentation: removed all `claude-flow` references, added full tool workflow (no agent cross-repo triggering attempted—focus is reproducer generation and manual handoff)

**Decisions:**

- Used single `tools/` crate with multiple `[[bin]]` targets rather than separate tiny crates—follows conventions of `cargo` and `ripgrep`, avoids dependency proliferation
- Kept Python script as reference; Rust version is the canonical tool going forward

**Status:** `turso-sql-replay` binary compiles cleanly with all 4 subcommands functional. Skill documentation complete and ready for reproducer workflow.
