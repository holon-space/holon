## Drag-Drop #10: IVM Hypothesis Disproven, Root Cause Still Open

**Context:** Earlier in this session I claimed open point #10's root cause was Turso IVM losing rows in `blocks_with_paths` on parent_id UPDATE. That claim was based on an inv10i diagnostic message that said "IVM MATVIEW INCONSISTENCY" with "1 row in matview, 3 expected visible". This continuation drilled into that to verify.

**What happened:**

- Wrote `crates/holon/examples/turso_ivm_update_parent_id_repro.rs` — minimal repro of the exact UPDATE pattern from `move_block`: 3 sequential UPDATEs (parent_id + 2 content) on a row that transitions from base case (parent_id LIKE 'doc:%') to recursive case in the `blocks_with_paths` materialized view.
- Ran the repro: **PASSES**. All 3 rows present after the UPDATE, including the moved row with its new path `/target/source`. So Turso IVM is NOT the culprit.
- Re-read inv10i in `sut.rs:3580+`. The "1 row" comes from the **root-layout query**, not the main panel. inv10i's `missing` check is even commented out as an explicit acknowledgment of the apples-to-oranges issue. So the IVM-corruption claim was based on a misread.
- The actual evidence is just inv16: 3 items in the live tree (`HeadlessLiveTree initialized on main panel (3 items, ...)`) but only 2 produce block_profile widgets in the rendered tree. The moved source goes missing.

**Challenges:**

- I jumped to an IVM conclusion too fast based on the soft-check message. Should have read the inv10i implementation before declaring root cause.
- Real root cause requires deeper instrumentation than fits a continuation session — needs eprintln!s or debugger-mcp breakpoints in `reactive_view.rs::Collection::build`, `entity_profile.rs::resolve_for_entity`, and `reactive_view_model.rs::to_view_kind` to identify which of the 3 items doesn't render a draggable and why.

**Decisions:**

- Updated `HANDOFF_DRAG_DROP.md` and the `drag_drop_wiring.md` memory entry to retract the IVM claim and document what's actually known.
- Kept the IVM repro file in `crates/holon/examples/` as evidence and as a future regression test for the recursive-CTE matview path.

**Status:** Drag-drop wiring is unchanged from the prior continuation; the false IVM hypothesis is removed; #10 is now correctly described as "root cause unconfirmed, need targeted instrumentation". The branch is in a stable state for handoff.
