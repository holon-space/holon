## Adding Pluggable Screenshot Capture to UI Tests

Screenshots were requested on every UI interaction, with element highlighting based on geometry bounds. The challenge: supporting multiple frontends (GPUI, Blinc) with different rendering architectures.

### What happened

- **Researched screenshot options** across frontends: Blinc has headless framebuffer capture (not yet wired up), GPUI has no widget-level offscreen rendering, so xcap window capture was chosen as the default
- **Designed `ScreenshotBackend` trait** — pluggable per-frontend strategy pattern, allowing future framebuffer backends without changing core code
- **Implemented `XcapBackend`** — finds window by title, captures via xcap, annotates with red highlight rectangle at element bounds using geometry coordinates
- **Integrated into `GeometryDriver`** — `with_screenshots(backend, output_dir)` enables capture; draws highlights using `image` crate's `put_pixel`
- **Hooked into test loop** — `driver.screenshot()` called after each step in `run_pbt_with_driver_sync`, with element info passed for UI operations
- **Updated dependencies** — added `xcap = "0.8"` and `image = "0.25"` to `holon-integration-tests`

### Challenges

- **xcap requires Screen Recording permission** on macOS (unavoidable for OS-level window capture; alternative of capturing GPUI's render tree would require forking GPUI to expose framebuffer readback)
- **Feature gate complexity** — since this is a test-only crate, moved `xcap` and `image` from optional to required dependencies and removed unnecessary `#[cfg]` gates
- **Headless test limitation** — xcap can't find a window titled "Holon" when test runs headless (no actual GPUI window spawned); screenshots still work when running against a real app window

### Decisions

- **Trait-based architecture** — allows swapping backends (e.g., `BlincFramebufferBackend` when Blinc wires up wgpu readback) without changing test code
- **Window title matching** — xcap requires finding windows by title; kept simple and configurable via `XcapBackend::new(title)`

### Status

Screenshot capture infrastructure implemented and compiling. Awaiting verification against actual running GPUI app to confirm window capture and annotation work correctly.
