## Phase 2 Tasks 2–4: Peer Persistence, QUIC Handshake, Worker Unification

**Context:** Following up on stable peer sync architecture, three cleanup bundles were prioritized: replacing the 300ms sleep-based teardown workaround with proper QUIC handshakes, eliminating the `OnceLock<Weak<Self>>` anti-pattern in `LoroShareBackend`, and unifying duplicated debounce logic between save and sync workers.

**What happened:**

- **QUIC handshake** (`iroh_sync_adapter.rs:78`): Added explicit `drain_until_eof()` on both initiator and acceptor. Initiator calls `send.finish()` then drains receiver; acceptor awaits `conn.closed()`. Removed the 300ms sleep entirely — race condition on QUIC teardown is now prevented by the protocol handshake, not timing.
- **`LoroShareBackend` construction** (`sync/loro_share_backend.rs`): Replaced `OnceLock<Weak<Self>>` with `Arc::new_cyclic(|self_weak| ...)`. Callback registration now happens at construction time without deferred state. Trait-dictated `&self` receiver in `SubtreeShareOperations` made this the cleanest fit.
- **Unified `DebouncedCommitWorker`** (`sync/debounced_commit_worker.rs`): Generic `spawn<F, Fut>(doc, filter, debounce, worker_name, work)` extracted from both `SaveWorker` (150ms, disk) and `SyncWorker` (500ms, network). Both now wrap it; filter helpers `any_commit()` and `local_only()` shared. Two new unit tests verify burst coalescing and filter semantics.
- **Shutdown flush wiring:** GPUI and MCP integrated; TUI/dioxus/flutter/others deferred (no single tokio main). GPUI uses `process::exit(0)` post-flush — pragmatic since `gpui_app.run()` blocks forever.
- **Miscellaneous cleanups:** `rand::thread_rng()` → `rand::rng()`; added `shared_tree::gc_orphans` op to enumerate mount nodes and delete orphan snapshot/peer/corrupt files; PBT cases bumped 16→24.

**Challenges:**

- `SaveWorker` error path now both emits bus event *and* returns `Err` — double signal is defensive but worth verifying intent.
- `OnceLock` elimination required `Arc::new_cyclic`, which is less familiar but avoids the deferred-registration smell.

**Status:** 84/84 sync tests passing; PBT 24 cases ~147s; `cargo check` clean on holon, holon-gpui, holon-mcp. Worktree ready for review before branch/commit isolation. Next: GPUI share/accept UI (#2 major feature), broadcast advertiser events (peer-online indicator prep), or stale-peer pruning.
