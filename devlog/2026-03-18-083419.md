## Configuring Holon with Unified Config Layer

Implemented a unified configuration system using the `premortem` crate to consolidate CLI, environment variable, and TOML file config sources with proper precedence and source tracking.

**What happened**

- Added `premortem` dependency and created `config.rs` module with `HolonConfig` struct supporting all Holon settings (orgmode, loro, todoist, UI, hooks)
- Implemented `ClapSource` to serialize parsed CLI args as a premortem source, enabling `Defaults → Toml → ClapSource` config pipeline
- Built `load_config()` to trace each config value to its source ("cli", "env:VAR", or "config.toml") for UI read-only locking
- Refactored `FrontendSession` to use new config types and added `is_preference_locked()` to check if a preference came from CLI/env
- Removed legacy API (`FrontendConfig`, `CliConfig`, `parse_args`, `build_frontend_config`) — now only `HolonConfig` + `SessionConfig` + `build_session()` exist
- Migrated all frontends (GPUI, Ply, TUI, MCP server) to new API; updated integration tests to construct config directly

**Challenges**

- **premortem API discovery**: Had to verify actual signatures (`ConfigValue::anonymous()`, `trace(&str)` not `&[str]`, `SourceErrorKind::Other` struct variant) by writing POCs
- **Config directory location**: Chose `~/.config/holon/` as platform-aware default, resolving `db_path` early so `holon.toml` loads before preference fallback
- **Locked keys extraction**: Avoided hard-coded preference path list by iterating all `TracedConfig::traces()` dynamically, checking which paths have source == "cli"
- **Test setup complexity**: Integration tests relied on `with_todoist_fake()` config method; migrated to test-specific DI closure (`extra_resolve`)

**Decisions**

- **Single source of truth**: One `HolonConfig` struct with both `clap::Parser` and `serde::Deserialize` derives, eliminating three separate types
- **Lazy DI resolution**: `new_from_config_with_extras` accepts closure for test-specific services, avoiding test-only config fields

**Status**: All 67 tests pass; full workspace compiles (excluding pre-existing Flutter auto-gen); config now follows CLI > Env > TOML precedence across all settings with automatic UI read-only locking for CLI/env sources.
