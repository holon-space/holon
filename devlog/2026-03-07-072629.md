## Extracted Shared Code Across Frontend Implementations

**Context:** Multiple frontend implementations (dioxus, waterui, gpui, blinc) contained identical or near-identical logic for CDC state management, event handling, and operation dispatching. These classes had no framework-specific code and could be shared through the `holon-frontend` crate.

**What happened:**

- Moved `CdcState` and `ui_event_listener` from dioxus to `crates/holon-frontend/src/cdc.rs` for reuse across callback-based frontends
- Moved operation helpers (`dispatch_operation`, `dispatch_undo`, `dispatch_redo`, `find_set_field_op`, `get_entity_name`, `get_row_id`) to `crates/holon-frontend/src/operations.rs`
- Eliminated 7 re-export shim files by updating all 40+ call sites across frontends to use `holon_frontend::` paths directly
- Removed 3 more shim files from `render/` directories (`context.rs`, `interpreter.rs`) after consolidating to holon-frontend
- Created `prelude.rs` modules in gpui and blinc render builders to eliminate repeated import boilerplate (3-4 lines → 1 line per file)

**Decisions:**

- Kept dioxus's local `dispatch_operation` (4-arg signature with ambient tokio) separate from the shared 5-arg variant used by waterui/gpui/blinc
- Did not move gpui's `AppState` or `cdc.rs` — uses `RwLock` + dirty flag pattern, fundamentally different from callback-based `CdcState`
- Created preludes in render builders to enable per-file widget definitions without import friction

**Status:** All workspace frontends compile cleanly. Shared code is now consolidated and per-file builder organization is frictionless.
