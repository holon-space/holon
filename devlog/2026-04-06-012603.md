## Minimal Inspector Widget Hierarchy

## Context
The GPUI inspector only provides single-element picking, not a widget tree view. We wanted to dump the GPUI entity hierarchy (BlockRefView → CollectionView → RenderBlockView, etc.) to the inspector without duplicating the render logic or refactoring all builders.

## What happened

- **Explored GPUI's inspector API** — `toggle_inspector`, `set_inspector_renderer`, `register_inspector_element` only track flat hitboxes and style for one selected element; no public tree walk API
- **Checked what data exists** — `DispatchTree` and `FrameState` contain the full element tree but are `pub(crate)` in Zed's GPUI
- **Evaluated separation approaches**:
  - Free Monad / Tagless Final (requires rewriting all builders generically — too large)
  - Render observer trait (requires threading through `GpuiRenderContext` — but that doesn't exist as a parameter)
  - Wrapping `Div` with traced versions (duplicate GPUI builder chains)
  - Reconstructing hierarchy from hitboxes (flat list, not a true tree)

## Challenges

- **No render context parameter** — `BlockRefView::render`, `CollectionView::render` take only `Window` and `Context<Self>`. Can't thread an observer through the signature without refactoring every entity's render impl.
- **Entity render roots are isolated** — each `Entity` renders independently; there's no single render "pass" where an observer could sit and see the full call stack.
- **Emergent structure** — the GPUI entity hierarchy (which things get their own `Entity`, which share one) isn't described in `ReactiveViewModel`; it emerges from rendering.

## Decisions

- **Focused on what's actually useful** — the entity-level hierarchy (which views own which, block IDs, item counts) plus per-element inspector (already shows bounds/style for picked element) is sufficient for debugging; full GPUI styling chains are visible in source
- **Deferred full tree capture** — would require either upstreaming a public tree API to GPUI, or refactoring entity renders to be generic over an algebra, both large changes

## Status

Inspector button works and shows single-element details; full widget tree dump would require architectural changes to how entity render methods are structured, so kept focused on leveraging what GPUI already exposes.
