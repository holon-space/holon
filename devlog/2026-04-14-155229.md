## QA of Responsive Layout (if_space) — Four Critical Bugs Found and Fixed

**Context:**
Phase 5 of the glittery-sprouting-lynx plan implemented a responsive `if_space` builder for three-column layouts across desktop/tablet/phone breakpoints. Manual QA revealed the feature wasn't working at all, triggering a deep investigation that uncovered four distinct bugs in the rendering pipeline.

**What happened:**

- Launched GPUI at desktop size (1400×900) and found the root layout rendering as a flat table instead of columns — traced to `if_space` missing from the Rhai DSL widget registry
- Added `"if_space"` to `render_dsl.rs` widget_names; rebuild verified columns rendered correctly
- Resized to tablet (900×700) but desktop branch still fired — discovered `interpret_pure` created a RenderContext with `available_space = None`, so `if_space` always picked the wide branch
- Added `BuilderServices::viewport_snapshot()` method and seeded it into both `interpret_pure` and `snapshot_resolved` paths
- Root was using `watch_data_signal` (ignored ui_generation bumps), so viewport changes never triggered re-interpretation — switched to `watch_signal`
- Tablet layout rendered but `if_col("region", …)` failed; GQL query doesn't expose a `region` column — changed to `if_col("content", …)` using the block heading text already in the row data
- All three breakpoints verified working: desktop showed three columns, tablet hid left sidebar (spacer), phone showed only main panel
- User reported left sidebar appeared to still occupy space when collapsed — added filter in `columns.rs` to skip zero-sized spacers from flex distribution (pending log verification)

**Challenges:**

- **Root context initialization gap**: Viewport data never reached the render DSL evaluation because the reactive engine created contexts with `available_space = None`; solution was to extract viewport via MCP and inject it into the BuilderServices
- **Signal type confusion**: Root re-interpretation required a signal that bumped on ui_generation changes, not just data changes; switching from `watch_data_signal` to `watch_signal` fixed the cascading reactivity
- **Missing column reference**: Phase 5's tablet render referenced a GQL column that didn't exist, causing silent failure; required understanding the actual query schema in `index.org`

**Status:**
Three of four breakpoints fully verified ✅. Spacer-compression issue requires log inspection to determine if the filter is correctly targeting the hidden slot element.
