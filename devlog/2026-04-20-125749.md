## Mount-node SQL projection and rehydration wiring

The session tackled the blocker preventing shared subtree mount rows from appearing in SQL after acceptance — mount metadata was stored only in Loro, never synced to the database where the UI queries it.

**What happened**

- Added `SHARE_ROLE_PROPERTY`, `SHARE_ROLE_MOUNT`, and `SHARED_TREE_ID_PROPERTY` constants to `shared_tree.rs`, using the `properties` JSON field instead of a new column to minimize blast radius
- Extended `LoroShareBackend` with optional `sql_ops: Arc<SqlOperationProvider>` field and `new_with_sql` constructor for dependency injection
- Implemented `project_mount_to_sql` helper that calls `execute_operation("block", "create", ...)`, triggering `EventKind::Created` so the cache subscriber refreshes and the UI sees the new row
- Wired projection into `accept_shared_subtree` after `save_all()` succeeds, and into the rehydration loop via new `MountRehydrationRecord` + idempotent `INSERT OR IGNORE`
- Updated `loro_module.rs` DI to resolve `DbHandleProvider` + `TursoEventBus` and inject `SqlOperationProvider` into the backend
- Fixed pre-existing STABLE_ID prefix bug in `set_stable_id`: it was storing `block:`-prefixed URIs while the rest of the codebase expected bare UUIDs, which would cause duplicates once SQL→Loro echo events started flowing
- Added missing `assets/default/notes.org` seed file (was breaking workspace build)

**Challenges**

- **Column vs. properties trade-off**: Adding `Block.external_id` column would require touching 40+ files. Used `properties` JSON instead to unblock the UI while keeping the diff small; migration to a dedicated column can happen in a later iteration.
- **STABLE_ID format inconsistency**: Discovered `accept_shared_subtree` was storing the `block:` URI prefix in metadata while downstream code expected bare UUIDs. Fixed the helper to strip the prefix before storage.

**Status**

Mount rows now project to SQL and rehydrate correctly across restarts (13/13 share backend tests pass, 3/3 sync PBT tests pass, iOS target builds). Descendant projection and live network updates deferred to next session.
