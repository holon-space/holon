## Seed Fresh Installation with Default Journals Document

Fresh installs showed an empty sidebar with no documents, making for a confusing first experience. We needed to ensure at least one document with content appears by default.

**What happened:**

- Created `assets/default/Journals.org` with a `* Journals` heading as the default seed file
- Updated `mobile.rs` to write `Journals.org` on iOS startup if it doesn't exist, using `include_str!` to embed the file
- Updated `frontend_module.rs` to write `Journals.org` on desktop startup if it doesn't exist
- Made seeding independent of directory emptiness check so it works on existing installs too

**Challenges:**

- Initial approach seeded `index.org`, but the sidebar PRQL filter explicitly excludes documents named `"index"` (after `.org` extension stripping by `path_to_name_chain`), so the document never appeared
- iOS simulator already had `holon-pkm/index.org` from a previous run, so the `is_empty` guard prevented `notes.org` from being written; fixed by making seeding unconditional on file existence rather than directory emptiness

**Decisions:**

- Used `include_str!` macro to embed the seed file as a single source of truth across both iOS and desktop paths, matching the existing pattern for `DEFAULT_INDEX_ORG`
- Named the seed file `Journals.org` instead of `index.org` to ensure it passes through the sidebar filter

**Status:** Fresh installs now show a "Journals" document on both iOS and desktop; seed file is centralized in `assets/default/` with compile-time embedding.
