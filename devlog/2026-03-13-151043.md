## Fixed "Org file diverged from reference" CDC cache race condition

The PBT test was failing because the org sync controller couldn't find newly-created documents in LiveData, causing duplicate document creation and orphaned blocks.

**What happened:**

- Investigated the `on_file_changed` panic about CDC cache not catching up — traced it to `get_blocks` BFS finding only 1 block instead of the expected 3
- Added extensive diagnostic logging to `LiveDocumentManager::find_by_parent_and_name` and discovered LiveData was never populated with child documents
- Found the root cause: **race condition in `LiveDocumentManager::create`**
  - `create()` inserts into the database synchronously
  - `find_by_parent_and_name()` reads from LiveData, which updates via CDC (eventual consistency)
  - Between insert and CDC delivery, callers see the document as missing and create duplicates
- Fixed by adding `LiveData::insert()` method and calling it directly after SQL insert (optimistic cache update)
- Also changed `assert!` → `bail!` in org sync controller to prevent fatal panics on CDC timeouts

**Decisions:**

- Used direct LiveData insertion rather than waiting for CDC to avoid the race — this is safe because we control both the SQL insert and the cache update
- Made the org sync controller resilient to CDC timeouts so one stalled CDC stream doesn't kill the entire sync loop

**Status:** "Org file diverged" errors eliminated; PBT now reaches later failure cases (display tree assertions unrelated to org file sync)
