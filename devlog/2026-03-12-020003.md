## Slash Command Menu for Block Operations

The goal is to enable a slash-command palette when typing `/` at the start of a line in editable text, allowing users to trigger operations like embedding blocks or changing block types. Currently, operation invocation logic is scattered across frontends (Flutter); it needs to be centralized in Rust.

**What happened:**

- Audited the codebase to understand operation architecture: `Operation` traits, `triggered_by` availability system, `ParamMapping`, and `TypeHint` for parameter resolution
- Reviewed existing `gpui-component` library which already provides `Input` editor and `CompletionMenu` widgets — these can be reused
- Checked Flutter frontend's `OperationMatcher` logic that resolves required params, identifies missing ones, and sorts by satisfiability — this needs porting to Rust
- Identified that `editable_text` currently receives only content-related operations; a convention change is needed so `editable_text` can trigger all row operations
- Planned phased implementation: core logic in `holon-frontend`, thin rendering in `@frontends/gpui/`, e2e tests in `general_e2e_pbt.rs`

**Decisions:**

- Centralize parameter resolution in `holon-frontend/src/operations/matcher.rs` (port from Flutter) — this handles satisfiability sorting, missing param detection, and entity type matching
- Broaden `editable_text` operation scope by convention so it receives all operations for its row, not just content-related ones — simplifies dispatch and UI logic
- Generalize `/embed` to support any entity type via `TypeHint::EntityId` — when params are missing, the UI shows an entity picker

**Status:** Architecture analyzed and implementation plan drafted; ready to begin coding the operation matcher and gpui integration.
