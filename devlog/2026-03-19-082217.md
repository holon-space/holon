## Fixed vertical scrolling in column layout by adding min-height constraints

The `column` view in the GPUI frontend wasn't responding to two-finger scroll events on macOS, despite having `overflow_y_scroll()` enabled in the code.

**What happened:**

- Analyzed the `columns.rs` builder and found all three code paths (drawer panels, non-edge drawer panels, non-drawer panels) already had `.overflow_y_scroll()` calls
- Checked the parent container hierarchy in `lib.rs` and confirmed `overflow_hidden()` was correctly set
- Realized the issue: **flexbox `min-height: auto` default** prevents flex items from shrinking below content size, making `overflow_y_scroll()` ineffective (container just grows to fit all content)
- Applied the standard Zed UI pattern: adding `.min_h_0()` to enable flex item shrinking
- Added `.min_h_0()` to 5 locations:
  - `lib.rs:333` — root content wrapper
  - `columns.rs:21` — columns container
  - `columns.rs:43` — drawer panels (sidebars)
  - `columns.rs:86` — non-edge drawer panels
  - `columns.rs:98` — main panel scroll container
- Verified compilation succeeds with no new errors

**Challenges:**

- The code *looked* correct at first glance since `overflow_y_scroll()` was present — the issue was CSS flexbox semantics, not missing scroll directives
- Root cause required understanding Taffy/flexbox behavior: `min-height: auto` on flex children prevents them from becoming the scroll container

**Decisions:**

- Used `.min_h_0()` rather than explicit `.min_h(px(0.0))` for consistency with Zed's codebase conventions

**Status:** Fixed and compiled. Ready for testing — two-finger scroll should now work on all three column paths in the layout.
