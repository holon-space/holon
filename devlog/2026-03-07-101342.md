## Ply Frontend Runtime Bringup & Handoff Completion

The Ply frontend was non-functional—text measurement used dummy values, the `live_query` builder was unimplemented, and the app crashed on launch. This session brought it to working state with live org file rendering.

### What happened

- **Text measurement** — Replaced dummy `Dimensions::new(0.0, 0.0)` with `Ply::new(&DEFAULT_FONT).await` using system Arial font. Text now measures correctly at render time.
- **live_query builder** — Wired the full chain: language detection → PRQL compile → context extraction → `query_and_watch()` → recursive `interpret()`, following Blinc's inline pattern.
- **Invisible window bug** — Discovered `runtime.block_on()` on the macroquad main thread during holon init blocked event processing. Restructured: show Ply window immediately with "Loading..." text, spawn holon backend on tokio thread.
- **White window bug** — Ply-engine's `show()` doesn't call `next_frame().await` internally. Added it after render loop; frame buffer now flips and displays correctly.
- **WatchHandle drop** — `_command_tx` from `watch_ui()` was being dropped at scope end, killing the UI watcher. Stored it in `HolonState` to keep alive.
- **MCP port conflict** — Changed default from 8520 (gpui using it) to 8521.
- **Runtime validation** — App launches, renders dark background, title bar, and live org file data (documents, headings, content with proper text sizing).

### Challenges

- **Async context mismatch** — Macroquad's `#[macroquad::main]` is async, but blocking on holon init froze the event loop. Solved by moving holon startup to background thread with loading UI.
- **Frame presentation** — Ply's renderer lacks `next_frame()` call; without understanding macroquad's frame-flip model, rendering silently failed. Resolved by reading ply examples and adding the call post-render.
- **Font loading** — No embedded font; used system Arial path (`/System/Library/Fonts/Arial.ttf`) on macOS. Noted for cross-platform work.

### Decisions

- **Layout rendered despite column overlap** — Prioritized getting something visible over perfect layout. Layout bugs are lower priority than proving the rendering pipeline works.
- **Async holon init** — Rather than restructuring holon's DI to be async-first, spawned it on tokio and polled for updates. Simpler and keeps holon isolated.

**Status**: Ply frontend renders live org file data with working text measurement. Column layout needs tuning; font needs cross-platform embedding. HANDOFF.md updated with remaining work.
