## Investigated and Fixed Semantic Bugs from Unused Parameter Sweep

The new `no_underscore_prefixed_params` architecture test revealed 205 violations. After triaging, we discovered three genuine semantic bugs hiding behind underscore-prefixed dead parameters.

**What happened:**

- Ran `no_underscore_prefixed_params` rule across the codebase and categorized 205 violations
- Identified 9 "genuinely suspicious" parameters (non-trait-stubs, actually used in some form)
- Investigated each one by reading function context and call sites
- Fixed 6 straightforward dead parameters (already cleaned up in current code)
- Discovered and fixed 3 real bugs

**Challenges:**

- **`_create_first_child` in `loro_backend.rs`** — The `true` branch once created default first-child blocks with full metadata (ID map, parent refs, timestamps) but this logic vanished during a refactor from map-based to LoroTree model. The bool parameter was left behind but ignored, meaning fresh document initialization silently stopped creating default content. Root cause: refactor removed data model but didn't reimplement the conditional branching. Fixed by collapsing both paths into one honest `initialize_schema` method.

- **`_expected_alpn` in `iroh_sync_adapter.rs`** — Looked like a security issue (ALPN negotiation parameter never validated), but investigation revealed ALPN filtering happens at QUIC handshake time via `builder.alpns()`, making the parameter redundant in the current single-ALPN-per-endpoint architecture. Fixed by removing the parameter and cleaning up 2 callers.

- **`exclude_origin` in `event_bus.rs`** — A builder method that claimed to filter by origin but silently discarded its argument (no-op). Zero callers, pure API lie. Fixed by deleting the method entirely.

**Decisions:**

- Collapsed `initialize_schema` and `initialize_schema_minimal` into one method since they had identical behavior — being honest about what the code actually does is better than maintaining a false distinction
- Removed `_expected_alpn` rather than "implementing" validation, since the architectural assumption (one endpoint = one ALPN) makes the check redundant

**Status:** All three bugs fixed. Architecture test violations reduced from 205 → 194 (and the remaining 194 are legitimate trait stubs/unimplemented methods that should use bare `_` instead).
