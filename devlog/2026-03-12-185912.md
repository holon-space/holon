## Implemented pie menu widget for GPUI with circular overlay layout

The pie menu widget needed a proper implementation beyond the flat-row TODO stub — operations should appear in a circle around the trigger element on right-click, rendered as a deferred overlay.

**What happened:**
- Studied GPUI's `deferred()` and `anchored()` primitives for overlay rendering without tree restructuring
- Added pie menu state tracking to `BoundsRegistry` with `toggle_pie_menu()`, `close_pie_menu()` methods
- Rewrote `frontends/gpui/src/render/builders/pie_menu.rs` with full circular layout implementation
- Right-click toggles menu open/closed; only one menu open at a time via exclusive state toggle
- Operations positioned on circle (radius 52px) using trigonometry, each in a 36px rounded button
- Click-outside dismiss via transparent backdrop at `deferred().with_priority(9)`, menu at priority 10
- Operation dispatch and menu close on button click; abbreviated long names to initials (e.g. "Mark Complete" → "MC")
- Added hover highlighting (accent color) and shadow for depth perception

**Challenges:**
- **Return type mismatch**: The trigger element became `Stateful<Div>` after adding `.id()` and `.on_mouse_down()`, but function signature required `Div`. Solved by wrapping in an outer `div()` container to satisfy return type while preserving event handling.
- **Theme color name**: Initial color reference didn't match the theme enum. Corrected to use proper accent color from theme.

**Decisions:**
- Used `deferred()` with priority ordering instead of restructuring the element tree — keeps the menu owner responsible for rendering its own overlay
- Exclusive toggle pattern (one menu at a time) via `BoundsRegistry` rather than per-element state — simpler to reason about and prevents competing menus

**Status:** Pie menu compiles and renders with right-click trigger, circular operation layout, and deferred overlay with proper z-ordering. Ready for testing.
