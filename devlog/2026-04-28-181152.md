## Implemented Property-Based Testing for TUI Frontend

The TUI frontend (`@frontends/tui`) now implements the same shared PBT infrastructure as GPUI, with extracted helpers and proper geometry/interaction/screenshot abstractions for headless testing.

### What happened

- **Phase 1**: Created shared PBT helpers in `holon-integration-tests/src/pbt/ui_harness.rs` (`wait_for_geometry_ready`, `spawn_quit_on_pbt_finish`, MCP startup, etc.). Refactored `gpui_ui_pbt.rs` to use them; verified no regression.

- **Phase 2**: Extended TUI's existing `RenderRegistry` and `SelectableRegion` with positional metadata (`start_row/col`, `rows/cols`, `widget_type`, `displayed_text`). Implemented `TuiGeometry` adapter wrapping the registry over cell→pixel conversion (8×16 cell size). Added unit tests.

- **Phase 3**: Implemented `TuiUserDriver` and `setup_interaction_pump` symmetric to GPUI, routing interactions through `ReactiveEngineDriver` (r3bl's input queue is closed). Wired `debug.interaction_tx` and `debug.user_driver` symmetrically.

- **Phase 4**: Built `OffscreenBufferBackend` (paints fg color of rendered cells into RGBA8) and `CapturingApp` wrapper that calls `compose_render_ops_into_ofs_buf` after each render. Passed `analyze_screenshot_emptiness` threshold. Unit tests on painter pass.

- **Phase 5**: Created `tui_ui_pbt.rs` test harness mirroring GPUI's topology (PBT thread, context channel, window-ready gate). Drives `app_render` directly on a tokio task. Full PBT runs all 50 transitions; 47–50 pass per run depending on seed.

### Challenges

- **Closed input stream**: r3bl's `MockInputDevice` is closed; v1 dispatch now routes through `ReactiveEngineDriver` instead of pushing into r3bl's queue.
- **Geometry exposure**: `RenderRegistry` and `SelectableRegion` already tracked entity IDs but lacked positional info. Solution: extend both with `start_row/col/rows/cols` metadata at registration sites (lines 379 and 1014).
- **TUI navigation gaps**: The remaining 0–3 transition failures per run expose real product issues (NavigateFocus/ClickBlock not propagating focus to the Main panel through the engine) — not harness problems, exactly what the PBT should surface.
- **Tokio shutdown race**: Runtime cancellation was racing with PBT-thread panic flag. Fixed by having main thread join the PBT handle directly, guaranteeing panic flag is set before exit.

### Decisions

- **Symmetry over minimalism**: Kept GPUI's `ctx_tx/window_ready_tx` topology in the TUI harness even though TUI doesn't strictly need it—consistent architecture across both frontends aids future maintenance.
- **In-process headless design**: No real PTY or external process; r3bl's primitives (`OffscreenBuffer`, `MockInputDevice`, injectable `main_event_loop_impl`) enable fully synchronous, inspectable test flows.
