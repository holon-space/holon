## Implemented Curried-Rolling-Truffle Plan & Fixed Cross-Block Navigation

### Context
Three planned improvements to the editor system were implemented, and a critical bug in cross-block navigation was discovered and fixed during investigation.

### What happened
- **ChildEntityManager extraction** — Created `child_entity_manager.rs` to consolidate ~80 lines of duplicated entity creation/GC/Drop code from `BlockRefView` and `RenderBlockView`. Both views now delegate block_ref, live_query, and editor reconciliation to shared methods.
- **Content diffing in RenderBlockView** — Added `structural_keys_match()` check in `set_content()` to skip full reconciliation when entity structure unchanged (data-only updates).
- **Virtual scrolling for CollectionView** — Integrated GPUI's `uniform_list` for collections >50 items; only visible rows render, reducing render cost from O(total) to O(visible).
- **Cross-block navigation restored** — Discovered and fixed a critical bug in `IncrementalShadowIndex::patch_block()` where node index adjustments after splice only updated nodes past the splice point. Parent nodes with stale child indices became unreachable, breaking the navigation tree structure. Fixed by adjusting ALL affected nodes, not just downstream ones.

### Challenges
- **Navigation debugging complexity** — The shadow index is built incrementally via patches; MCP's debug cache initially appeared stale (showing 0 editor inputs) because it wasn't updated on every patch. Realized the shared `Arc<RwLock<ShadowIndex>>` meant the cache was accurate; the real issue was structural corruption in the tree.
- **Index adjustment logic** — The splice adjustment loop in `patch_block()` had subtle off-by-one implications. Required careful analysis of how node indices shift when a block's content array grows mid-operation.

### Decisions
- **Keep ChildEntityManager as plain struct** — Not a Component; it's owned by `BlockRefView`/`RenderBlockView` and managed via Drop impl for cleanup.
- **Defer full shadow index diagnostic tools** — Added targeted tracing in `handle_cross_block_nav` and `bubble_input` rather than expanding MCP tooling further.

### Status
All three plan items implemented and compiling cleanly. Cross-block navigation (cursor Up/Down between editable blocks) is restored. Left minimal tracing in place for future navigation debugging.
