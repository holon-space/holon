## Enhanced `turso-sql-replay` Tool with Detailed Matview Diffing

**Context:** The SQL replay tool needed better visibility into materialized view inconsistencies. The origin script had been improved with a detailed EXCEPT-based diff comparison that shows exactly which rows are extra or missing.

**What happened:**

- Ported the enhanced `check_matview_consistency` logic from the origin `turso_sql_replay.rs` to the repo's `tools/sql-replay/main.rs`
- Key improvement: when a mismatch is detected, the tool now creates a fresh temp matview and runs two EXCEPT queries to isolate problematic rows
- Added explicit output showing `EXTRA rows in matview` and `MISSING rows from matview` with actual data
- This replaces the previous approach of just reporting "row counts don't match" with actionable diff data
- Verified the tool builds cleanly and runs on the minimal reproduction SQL

**Decisions:**

- Kept the tool's core flow unchanged — only enhanced the consistency check function to match the origin's diffing approach
- Used temp table `_fresh_mv` to avoid polluting the main schema while preserving the original matview for inspection

**Status:** `turso-sql-replay` now provides detailed row-level diffs when IVM inconsistencies are detected, enabling faster root cause identification during bug investigation.
