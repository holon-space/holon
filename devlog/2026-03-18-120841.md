## Plan: Refactor Frontend Architecture & Clean Up

**Context**: Following the ViewModel migration, we need to finalize the architecture by eliminating dead code, centralizing interpretation logic, and minimizing the public API surface.

**High-level approach**:
1. Delete dead code (`CdcState`, `ShadowDom`)
2. Move interpretation responsibility into `FrontendSession` 
3. Ply: Re-adopt `builder_registry!` macro pattern instead of hand-written match
4. Minimize public API: hide `RenderPipeline`, `RenderInterpreter`, `BlockWatchRegistry` from frontends
5. Remove `preferences_render_data()` — let frontends read preferences themselves
