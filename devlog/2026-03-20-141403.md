## Cross-block cursor navigation (Up/Down at block boundary)

**Context**
The GPUI frontend lacked cross-block navigation when the cursor reached a block boundary. Moving up at the first line or down at the last line would silently stop instead of transferring focus to the adjacent block while preserving column position, as the `holon-frontend` ShadowDom infrastructure already supported.

**What happened**

- Patched the `gpui-component` fork to propagate MoveUp/MoveDown actions at block boundaries (following Zed's pattern): `up()` and `down()` now call `cx.propagate()` when the cursor doesn't move, instead of silently stopping.
- Added `set_cursor_offset()` public method to InputState to allow cursor positioning without requiring `&mut Window`.
- Removed boundary-blocking logic from `EditorView` capture_action handlers for MoveUp/MoveDown, allowing actions to propagate naturally.
- Extended `BoundsRegistry` with `shadow_dom` and `editor_inputs` storage, plus helper methods for navigation: `bubble_input()`, `focused_editor_row_id()`, `focused_cursor_column()`.
- Built `ShadowDom` during `AppModel::rebuild()` from the view model, and registered InputState entities keyed by row_id after editor reconciliation.
- Added parent-level `on_action` handlers in `HolonApp::render()` for MoveUp/MoveDown that intercept propagated actions, compute the target block via `ShadowDom::bubble_input()`, convert placement to cursor offset, and transfer focus + position in one atomic operation.
- Pushed gpui-component changes to the `holon` branch and verified `cargo update` resolves to commit `5ce62689`.

**Status**
Cross-block navigation infrastructure complete. `cargo check -p holon-gpui` compiles cleanly; `cargo test -p holon-frontend` passes all 67 tests. Ready for manual testing with the running app.
