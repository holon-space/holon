## Fixing Reactive Bridge Between Turso Backend and GPUI Frontend

After refactoring to introduce futures-signals between Turso and the GPUI frontend, the UI rendered nothing despite the backend processing data correctly.

**What happened**

- Diagnosed backend logs showing `watch_ui` succeeding with correct data (root-layout columns, 3 items)
- Found first bug: `try_send` on bounded(1) channel dropped real ViewModels when channel still held initial empty value, preventing signal from emitting again
- Fixed by replacing `try_send` with `send().await` to block until consumed
- Encountered panic in `BlockRefView::new` calling `tokio::runtime::Handle::current()` from GPUI context (outside tokio runtime)
- Fixed by threading `rt_handle` through `EntityRegistry` instead of calling `Handle::current()`
- Discovered `describe_ui` MCP was hardcoded to return empty data, making it useless for debugging
- Refactored MCP to share the same `ReactiveEngine` as GPUI by creating it once in `main.rs` and passing via DI
- Fixed editor reconciliation: `walk_editors` wasn't recursing into `BlockRef` subtrees from async `BlockRefView` components
- Added `current()` accessor to `BlockRefView` and updated `reconcile_editors` to walk both root and async live-block ViewModels

**Challenges**

- **Channel backpressure + signal deduplication**: Initial empty signal persisted in bounded channel, blocking subsequent real data. Solution required understanding that `send().await` waits for consumption before next poll coalesces updates.
- **Runtime context mismatch**: GPUI context isn't a tokio runtime, but reactive bridge tried to spawn tasks. Solution: pass handle explicitly through DI rather than calling current().
- **Asynchronous subtree rendering**: Root ViewModel contains empty live-block nodes; real data lives in separate `BlockRefView` signals. Required walking both trees during reconciliation.

**Status**

Backend and MCP now share reactive engine with correct data. GPUI window still renders blank—likely signal not firing after recent refactoring; needs tracing to verify stream initialization in new architecture.
