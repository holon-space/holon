## Fix org file discovery to recursively scan subdirectories

The org sync provider was only scanning the top-level directory for `.org` files, causing `Projects/Holon.org` and other nested files to be missed. The docstring claimed to use the `ignore` crate, but the implementation was just a flat `read_dir()`.

**What happened:**
- Identified that `scan_directory` in `file_watcher.rs` only reads top-level entries with `std::fs::read_dir(root)` — no recursion
- Verified `ignore` crate was already a workspace dependency
- Replaced flat iteration with `ignore::WalkBuilder` which:
  - **Recursively** traverses subdirectories
  - Respects `.gitignore` files and skips hidden dirs (`.git`, `.jj`)
  - Automatically filters out non-regular files
- Tested compilation — clean build

**Status:** `Projects/Holon.org` and other nested org files will now be discovered and ingested. The ClaudeCode.org UI rendering issue remains separate — data ingestion works fine, the problem is GPUI not re-rendering after the root view model updates asynchronously.
