## V1: MCP `scroll` tool — observable semantics change

**Context:** The previous session's fail-loud refactor changed `GpuiUserDriver::scroll_entity` to return `Err` when element bounds aren't recorded. This surfaces through the MCP `scroll` tool at `frontends/mcp/src/tools.rs:1684-1722`, which converts driver errors into `rmcp::ErrorData::internal_error`.

**Observable change:** Before: `scroll { entity_id: "bogus" }` returned `Ok` with a silent no-op inside the driver. After: the same call returns an `internal_error` carrying the bounds-missing context string ("no bounds recorded for entity... element not rendered, or BoundsRegistry hasn't promoted staged → committed since it was added").

**Audit result:** Zero in-tree callers rely on the old silent-Ok behavior. The MCP tool description never promised silent success. Test fixtures and integration tests do not call the scroll tool. The change only affects direct MCP clients that might have been ignoring error responses.

**Action taken:** None required beyond this disclosure. Any external MCP client should read tool error responses and surface them; this change adheres to that contract.
