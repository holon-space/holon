## Integrate gpui-component Editor into source display

The `@frontends/gpui/` frontend was using raw `div()` primitives for displaying code, missing out on the richer `Editor` widget available in the `gpui-component` dependency.

- Analyzed current `source_editor.rs` builder structure and identified that builders return `Div` without access to `Window`/`App` context needed for `InputState`
- Designed a `RenderOnce` wrapper component that defers `InputState` creation to render time when context is available, then wraps it in `Component::new()` for `IntoElement` compatibility
- Updated `source_editor.rs` to replace raw text `div()` with `gpui_component::input::Input` configured as a code editor with line numbers and syntax highlighting
- Applied the same pattern to `source_block.rs` for consistent source content display while preserving the header (language label, name, run button)
- Verified full compilation with `cargo check`

**Challenges:**

- The builder pattern returns `Div` without GPUI context available — solved by deferring state creation to render time via `RenderOnce`
- `RenderOnce` doesn't auto-impl `IntoElement` in GPUI 0.2 — needed explicit `Component::new()` wrapper to satisfy type system

**Decision:**

Used deferred rendering pattern rather than restructuring the builder API, minimizing changes to existing architecture while gaining full editor capabilities.

**Status:** Both `source_editor.rs` and `source_block.rs` now use gpui-component's code editor; compiles cleanly.
