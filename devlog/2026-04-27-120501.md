## Extended Reproducer & MCP Instrumentation for Cross-Region IVM Churn

The `gpui_ui_pbt` test flagged a different inv16 shape than the Turso fix addressed: spurious 4-item CDC batches on `region:main` after clicking in `LeftSidebar`. The original reproducer only tested same-region navigation, so it missed this.

**What happened:**

- Extended [`turso_ivm_focus_roots_churn.rs`](crates/holon/examples/turso_ivm_focus_roots_churn.rs) with three phases: (1) original empty-batch churn, (2) cross-region navigation scenario, (3) `editor_cursor` writes
- All three phases passed — ruling out cross-region IVM filter leakage and unrelated table writes as causes
- Identified that the `gpui_ui_pbt` failure must come from a production-specific side effect not modeled in the reproducer
- Created a configurable pause/inspection hook system for PBT tests to enable MCP attachment during failures

**Challenges:**

- The leak wasn't cross-region filter churn or `editor_cursor` CDC — required deeper instrumentation to identify the real upstream write
- PBT tests need deterministic stopping points for live debugging; designed `PBT_PAUSE_BEFORE_STEP` / `PBT_PAUSE_AFTER_STEP` / `PBT_PAUSE_SECONDS` env vars plus richer inv16 diagnostics (dumping actual spurious rows)

**Decisions:**

- Made pause hook reusable across all PBT variants (headless integration tests and GPUI UI tests) rather than one-off
- Added PID to banner for easy process inspection with MCP
- Enhanced inv16 assertion to dump full row details instead of just counts

**Status:** Reproducer phases 1–3 are clean. MCP instrumentation ready; next step is running `gpui_ui_pbt` with pause hooks enabled to capture the failing transition state and identify the real source of the 4-item spurious batch.
