## Obsidian Vault Integration & Cross-System Task Ranking

**Context:** Holon has strong architecture (PRQL queries, sync, MCP) but needs a compelling first release that overcomes skepticism about yet-another-PKM tool. Research shows users want cross-system visibility without migration.

**What happened:**
- Analyzed implementation status: editor (working), Todoist sync (working), org-mode (working), render DSL (partial), MCP server (working)
- Identified critical gaps blocking adoption: Obsidian vault sync, visual widgets (calendar/Kanban), editor polish, quick capture
- Mapped user skepticism vectors: "just another tool," "vaporware," "AI slop," "will it replace my workflow"
- Designed three-tier feature release: MVP (cross-system queries + Obsidian sync), Tier 2 (visual widgets), Tier 3 (GQL/Petri Net)
- Created concrete feature list for first release, prioritized by impact on user adoption vs. implementation cost

**Key findings:**
- Both Obsidian and LogSeq expose file-level access (no app required) — zero-friction integration path already available
- "Keep your existing PKM, add Holon for queries" is the positioning that overcomes migration friction
- Top 3 user pain points across both communities: (1) scattered tasks across tools, (2) no cross-system prioritization, (3) table/calendar visualization missing from text-based tools
- Editor correctness (no data loss, responsive UI) is table-stakes; polish can come later
- WSJF ranking across Obsidian + Todoist + JIRA is a differentiator no other tool offers

**Decisions:**
- Prioritize Obsidian vault sync (file-level watcher) over LogSeq initially — larger user base, simpler format
- Lead with "unified task view + WSJF ranking" in marketing, not "new PKM system"
- Ship calendar widget before Kanban (lower implementation cost, higher weekly-review workflow impact)
- Accept that first release has basic editor — focus on zero data loss and 60fps, not vim mode

**Status:** Feature prioritization complete. `docs/FIRST_RELEASE_FEATURES.md` links from MARKET_RESEARCH.md. Next: Begin Obsidian vault watcher implementation.
