## Fixed `WorktreeCreate` Hook to Support Workspace Resume

The `WorktreeCreate` hook in `.claude/settings.local.json` was failing when resuming an existing workspace, as the hook would attempt to re-run initialization steps on directories that already existed.

**What happened:**
- Identified that the hook was executing unconditionally on both fresh creation and resume operations
- Made the hook idempotent by adding an existence check at the start
- If the worktree directory (`$cwd/.claude/worktrees/$name`) already exists, the hook now echoes the path and exits cleanly
- Fresh workspace creation still runs the full initialization: `jj workspace add` and `cargo check`
- Resume operations now work correctly whether or not Claude re-fires the hook on context restoration

**Decisions:**
- Chose idempotency over conditional hook registration because it's simpler and handles both cases transparently — the hook is safe to fire multiple times without side effects or errors

**Status:** Hook now supports both fresh workspace creation and resuming existing workspaces without errors.
