## Debugging EditableText Blur Event Bug

The `editable_text` widget in `@frontends/gpui/` was not sending operations to the backend when text was modified and the field lost focus, despite blur event handlers being present in code.

**What happened:**

- Started a debug session with LLDB via debugger_mcp, targeting the Holon app binary with debug profile
- Set strategic breakpoints at `HolonApp::render()` (line 42), `ensure_input_states()` collection (line 491), and the blur event handler (lines 513, 520)
- Confirmed `HolonApp::render()` was executing but the `ensure_input_states()` condition at line 48 (`self.cached_display.is_none() || self.spec_dirty.swap(false, ...)`) evaluated to `false` on re-renders
- Stepped through render logic and confirmed `cached_display` was `Some` and `spec_dirty` was `false`
- Waited for blur event breakpoint to fire after user interaction (click into editable field, type, blur) — **breakpoint never triggered**

**Challenges:**

- Initial breakpoints on `ensure_input_states` weren't firing because the function only runs when `spec_dirty` is true or cache is empty. GPUI input keystroke redraws don't trigger re-render, they use cached display tree
- Discovered two code paths for creating editable_text widgets: the `widget_builder!` macro path (which includes operations and entity IDs) vs. the fallback `ViewModel::element()` path (line 20-28 in `render_entity.rs`)
- The fallback path creates nodes with **empty entity map and empty operations vector**, preventing InputState entity lookup and operation dispatch from working

**Root cause identified:**

When `render_entity.rs` creates a `TextContent` editable_text node via `ViewModel::element("editable_text", ...)`, it bypasses the `widget_builder!` macro and creates a node with no entity ID and no operations. This causes `ensure_input_states()` to skip subscription setup, so blur events never trigger `dispatch_operation()`.

**Status:** Bug root cause isolated — the `render_entity.rs` fallback path needs to pass operations and entity ID when creating editable_text nodes, matching the `widget_builder!` macro behavior.
