## iOS Software Keyboard Not Pushing Content Up

The text input field now correctly shows the iOS software keyboard when focused, but the app window doesn't resize and the focused input gets hidden behind the keyboard.

**What happened:**

- Investigated how gpui-mobile provides keyboard height via `gpui_mobile::keyboard_height()`, which is polled each frame and updated by iOS `UIKeyboardWillShow`/`UIKeyboardWillHide` notifications
- Discovered the keyboard height flows through `IosWindow`'s `TEXT_INPUT_DIRTY` flag, triggering re-renders via CADisplayLink
- Added keyboard height to `safe_area_bottom_px()` in `mobile.rs` using `max(safe_area_inset, keyboard_height)` on iOS
- Found that this padding change doesn't actually shift the content because the reactive layout system uses `AvailableSpace` (via `apply_viewport`) rather than GPUI padding for layout decisions

**Challenges:**

- iOS doesn't resize the window when the keyboard appears—it covers content in-place. Unlike Android's `adjustResize`, iOS requires explicit handling
- GPUI padding adjustments don't flow through to the reactive signal-based layout system that `HolonApp` uses via `apply_viewport`
- The keyboard height change is asynchronous (iOS notification) but properly triggers re-renders via `TEXT_INPUT_DIRTY`

**Decisions:**

- Reverted the padding-only approach; the real fix needs to adjust viewport height via `apply_viewport()` rather than GPUI padding
- Plan: track `last_keyboard_height` in `HolonApp` and call `apply_viewport()` with reduced available height (`window_height - keyboard_height`) when it changes

**Status:** Keyboard shows and accepts input, but content layout doesn't shift. Need to implement viewport height adjustment in `HolonApp::render()` to properly reflow the reactive layout.
