## Migrated render DSL from s-expression to Rhai syntax

The render DSL was recently extracted from PRQL into a separate Rhai-based language, but test files still contained old s-expression syntax that the parser no longer accepts.

**What happened**

- Identified render DSL parse errors (`Unexpected 'item_template'`) failing both `gpui_ui_pbt.rs` and `general_e2e_pbt.rs` harnesses
- Migrated 3 legacy s-expression `list item_template:(...)` strings in `generators.rs` to new syntax: `list(#{item_template: row(...)})`
- Stripped 10 dead `render (...)` PRQL stages from `bidirectional_sync.rs` (render no longer exists in PRQL; was leftover code)
- Updated 2 render strings in `turso_ivm_index_bug.rs` and 4 in `e2e_backend_engine_test.rs`, with `{{}}` escaping applied in format strings
- Removed 1 render doc comment from `holon-todoist/src/queries.rs`
- Verified all changes compile; confirmed parse errors gone from test output

**Challenges**

The render DSL fallback to `table()` masked downstream failures — once the new syntax rendered correctly, pre-existing bugs in MCP sync and org-sync block-tracking surfaced in PBT runs. The non-determinism in `general_e2e_pbt` makes them occasional rather than consistent.

**Status**

Render DSL migration complete across codebase. Parse errors eliminated. Build clean. Pre-existing bugs (`mcp_sync_engine.rs:303` overflow, org-sync block-not-appearing) remain and are tracked separately.
