## Continuing iOS GPUI Frontend — Safe-Area Insets, Seeding, and Merge Readiness

### Context
Holon's GPUI iOS simulator frontend was rendering theme chrome but with layout issues (status bar overlap, empty content on first launch). This session focused on completing the remaining polish tasks and verifying the changes are safe to merge back to main.

### What happened

- **Safe-area insets wired (task #1)**: Added `safe_area_bottom` field to `HolonApp`. Updated `HolonApp::render` to query both top and bottom insets every frame on iOS/Android via `safe_area_top_px()` and new `safe_area_bottom_px()` helpers in `mobile.rs`. Applied `.pb(px(safe_area_bottom))` to root page div. Verified on simulator: status bar (clock/notch) now sits cleanly above title bar; home-indicator area no longer overlapped.

- **iOS first-launch seed implemented (task #2)**: Modified `ios_data_paths()` in `mobile.rs` to write embedded `assets/default/index.org` into `Documents/holon-pkm/index.org` when the directory is empty on first launch. Confirmed file appears on disk in the app sandbox. Empty columns on screen are correct — the seed contains layout chrome only, no user content blocks.

- **Debug eprintln cleanup (task #3)**: Removed `[theme]` debug print from `apply_holon_theme`. Kept the ops-useful `GPUI iOS: db_path=…` diagnostic in `ios_data_paths()`. Removed stray eprintln from reactive shell.

- **Build verification**: `cargo check -p holon-gpui` (desktop) and `just ios-build` (aarch64-apple-ios-sim) both succeed cleanly. Lib unit tests compile. Pre-existing test breakages (`Point::height` field mismatches, function-arity errors in `layout_scroll.rs`, `refresh_cascade.rs`) are present on main itself — not regressions from this session.

- **Merge-back risk assessment**: Scope is isolated and gated. Desktop path unaffected by iOS cfg blocks. The `launch_holon_window_impl` refactor (cx.spawn → synchronous + foreground_executor().block_on) affects the PBT pre-warm path; functional verification requires unblocking pre-existing test breakages first (out of scope).

### Challenges

- **PBT verification**: The property-based test suite can't currently verify the pre-warm codepath because `tests/layout_proptest.rs` and related files fail to compile on main itself. The regression risk is real but can only be validated after those tests are fixed upstream.
- **Pre-existing UAF in gpui-mobile**: The `touches_ended` crash (raw pointer list without ownership) surfaced during manual testing but is a separate known issue flagged for upstream PR #3. Did not address in this session.

### Decisions

- **Per-frame inset refresh over initialization**: Chose to query safe-area insets every frame in `render()` rather than plumb them into `HolonApp` at init. This handles rotation and split-view changes if gpui-mobile ever supports them.
