## Persistent `watch_ui()` + Shadow DOM Interpretation in PBT

**Context:** The PBT tested invariant 10 by calling `render_block()` directly—a one-shot API that real frontends never use. Production frontends call `watch_ui()` once and maintain the `WatchHandle`, receiving `UiEvent::Structure` on re-renders and interpreting them into `ViewModel` trees via the shadow interpreter.

**What happened:**

- Added `try_recv()` method to `WatchHandle` in `holon-api/src/streaming.rs` for non-blocking event draining
- Added `count_error_nodes()` helper to `display_assertions.rs` and fixed pre-existing `ViewModel::EMPTY` references
- Added `root_watch: RefCell<Option<WatchHandle>>` field to `E2ESut` with constructor initialization
- Replaced invariant 10 entirely: persistent `watch_ui()` → drain pending events with `try_recv()` → interpret latest `WidgetSpec` via `create_shadow_interpreter()` → verify `ViewModel` tree
- Implemented take-put pattern around `.await` points to avoid holding `RefCell` borrow across async boundaries
- First check block-waits for initial Structure event; subsequent checks skip if no re-render occurred

**Challenges:**

- **`RefCell` across `.await`:** `RefMut` isn't `Send`, so can't hold the borrow during async operations. Solved by extracting watch with `take()`, working with it, then `put()` back before any `.await`.
- **No new Structure events on unchanged blocks:** Initial timeout panic occurred because after first Structure, subsequent calls to `try_recv()` got nothing, then blocked waiting for a Structure that never came. Fixed by only block-waiting on newly-created watches; subsequent calls gracefully skip if the block hasn't re-rendered.

**Decisions:**

- Use `RefCell` for interior mutability since `check_invariants` runs single-threaded inside `runtime.block_on()` 
- Drain *all* pending events and use the last Structure event to mirror real frontend behavior—they always render the latest state
- Log counts of received Structure events (22 times) vs. skipped checks (28 times) to show watch lifecycle is working

**Status:** Invariant 10 now exercises the full production path: `watch_ui()` → `UiEvent::Structure` → `WidgetSpec` → shadow interpret → `ViewModel` tree. Compiles cleanly, zero inv10-related panics in test runs.
