## Audit and Fix for Block Split Input Routing

Splitting a block via Enter key in the GPUI editor was broken because the multi-line input component was consuming the keystroke without dispatching the associated action.

**What happened**

- Diagnosed root cause via log analysis and code inspection: `InputState` in `editor_view.rs` is configured as multi-line (`.auto_grow(1, usize::MAX)`), so `InputState::enter` inserts `\n` and never propagates, preventing `on_action(Enter)` from firing
- Fixed in `editor_view.rs` by dispatching `split_block` directly in the `EditorAction::None` branch, mirroring the existing Tab→indent pattern
- Migrated PBT infrastructure to eliminate synthetic dispatch shortcuts:
  - Rewrote `general_e2e_pbt.rs` block ops to route through `send_key_chord` instead of direct `dispatch_block_op`
  - Updated `ReactiveEngineDriver::send_key_chord` to build focus paths via `engine.snapshot_reactive`, eliminating the `HeadlessInputRouter` timing gap that justified the original bypass
  - Removed short-circuit in `GpuiUserDriver::send_key_chord` for `extra_params`; now positions cursor via real keystrokes (home + right arrows)
- Audited every remaining backend-bypass site; added `TODO(simulate-real-input)` comments and created tracking tasks #8–#14

**Challenges**

- **PBT baseline red on unrelated failures**: `gpui_ui_pbt` panics on peer-merge assertions and timing issues in the working copy's WIP `loro_block_operations.rs` changes; couldn't verify the fix end-to-end on GPUI window. Parked as tasks #15–#16.
- **Misconceived #10**: Initially tried to route `MutationSource::Action` through keychords, but actions are autonomous backend dispatches with no user keystroke to simulate. Reverted and clarified scope.
- **UI fallback pretends at gestures**: Audit of `sut.rs:4839` revealed that many mutations (delete, arbitrary create, pref updates) claim to simulate user input but have no actual UI affordance. Reclassified these as pretend operations and flagged for design decision.

**Decisions**

- Keep deleted `dispatch_block_op` call sites on a burn-down list rather than force-migrate all at once; allows incremental review
- Implement strict mode (`PBT_STRICT_INPUT=1`) to panic on synthetic fallback instead of silently allowing it, making gaps visible without breaking existing tests
- Block #9 (content-update mutations) pending design of a `replace_text(entity_id, new_text)` UserDriver verb

**Status**

Bug fixed in `editor_view.rs`; headless PBT chord routing now uses real input pipeline (no more synthetic dispatch shortcut). Burn-down tasks #8–#14 tracked. User notes stale content displayed for original block after split — parked as #17 pending PBT baseline fix.
