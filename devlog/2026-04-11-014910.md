## Improved Plan: Unify Collection Rendering on a Single Stream (v2)

### Context

The GPUI frontend shows `(empty)` under every `block_ref` region because `ReactiveShell::new_for_block` calls `reconcile_children` once at construction against an empty `MutableVec` — it never subscribes to the streaming driver that fills it later. Additionally, `collection_with_source` has a footgun: when both `data_source` and `item_template` are present, it discards the template and creates an empty collection, then relies on async timing to populate it.

### What happened

- **Diagnosed the actual bug:** `ReactiveShell` doesn't listen to nested collection signal_vecs over their lifetime; it only snapshots at construction.
- **Split into four sequential PRs** to isolate the critical fix from mechanical refactoring:
  - **PR1:** Add `ReactiveShell.collection_subs: Vec<AbortHandle>` + subscribe to inner `view.items` streams on construction and structural rebuild. Explicitly `.abort()` handles on tree changes; don't rely on `Drop`. This alone should fix the blank-UI symptom.
  - **PR2:** Introduce `CollectionData` enum (`Streaming { item_template, data_source }` | `Static { items }`) in macro expansion and shadow builders. Eliminates the "discard items when data_source present" footgun. No runtime behavior change.
  - **PR3:** Migrate `HeadlessBuilderServices` and snapshot paths to use `ReactiveQueryResults` instead of raw `data_rows`, allowing full `Streaming` path.
  - **PR4:** Fold `columns` sort/drawer logic into the streaming driver (deferred, depends on PR1-3 stability).
- **Made concrete the footguns from original review:** re-entrant reconcile guard, `AbortHandle` explicit abort, stale diff handling, `auto_body` audit (grep required before PR2 opens), and dead-code confirmation of `(Some, None)` arm via test fixture inventory.

### Challenges

- **Re-entrant reconcile risk:** Added a `bool` guard on `ReactiveShell` to early-return if `reconcile_children` is already in progress, with a test that mutates a row during reconcile to catch stack-overflow.
- **PR1 test must fail on main:** Explicitly required that a GPUI integration test be added that fails before PR1 and passes after, per CLAUDE.md rule #1. Blocker for merge.
- **`(Some, None)` arm legitimacy:** Original plan treated it as dead, but it's hit by headless/stub paths. Concrete confirmation owed: a grep of `interpret_fn.*data_rows` to enumerate which test fixtures still use it, so PR3 can migrate them.

### Decisions

- **Four PRs instead of one:** PR1 is the minimum viable fix and ships first; if it solves the visible bug, PR2-4 proceed as cleanup/correctness rather than load-bearing. Easier to bisect if something regresses.
