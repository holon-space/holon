## Unified logging implementation for GPUI and TUI

## Context
The frontends had separate logging configurations (`LOG_DEST` in GPUI, inline setup in TUI) with no support for structured OpenTelemetry output. The goal was to unify on a single `HOLON_LOG` env var that routes to appropriate formats (human-readable for files/stdout, JSON for OTel).

## What happened

- **Added OTel dependencies to workspace** (`opentelemetry-otlp`, `opentelemetry-stdout`, `tracing-opentelemetry`, `opentelemetry-appender-tracing`) behind `otel` feature flag in `holon-frontend`
- **Rewrote `crates/holon-frontend/src/logging.rs`** to implement Option C: single `HOLON_LOG` env var that routes to natural output formats
  - `stderr` / `stdout` → `fmt::layer()` (human-readable)
  - `file:///path` → `fmt::layer()` with file writer, no ANSI codes
  - `otlp` → `tracing_opentelemetry::OpenTelemetryLayer` + OTLP exporter
  - Supports comma-separated multiple targets: `HOLON_LOG=stderr,otlp,file:///tmp/holon.log`
  - Default: `stderr`
- **Updated GPUI** (`frontends/gpui/src/main.rs`): already called `init()`, just removed stale comments. Now gets OTel support via feature flag
- **Refactored TUI** (`frontends/tui/src/main.rs`): replaced ~20 lines of inline logging with centralized `holon_frontend::logging::init()`. Defaults to `file://~/.config/holon/tui.log` (preserves terminal), respects explicit `HOLON_LOG` env var
- **Enabled `otel` feature** in both GPUI and TUI `Cargo.toml`

## Challenges

- **Type system complexity with dynamic layer composition**: Initially tried using `Vec<Box<dyn Layer<Registry>>>` combined with `.with(EnvFilter)`, but the filter changes the subscriber type to `Layered<EnvFilter, Registry>`, causing type mismatches. **Solution**: moved `EnvFilter` inside the layers vec so all layers are parameterized on the same `Registry` type.

## Status

Both GPUI and TUI now support unified logging via `HOLON_LOG` env var with format-aware routing. All OTel infrastructure (OTLP exporter, stdout exporter) is wired up and working. Both frontends compile cleanly.
