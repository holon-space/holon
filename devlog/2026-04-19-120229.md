## Step 8 Completion, Mobile Action Bar, and Task #12 Concat Migration

Continued the v3 rendering infrastructure plan with focus_chain value function, mobile action bar YAML integration, and concat migration to registered ValueFn.

**What happened**

- Implemented `focus_chain()` value fn in `value_fns/focus_chain.rs`: subscribes to `UiState.focused_block_mutable()` signal, emits rows representing focused block at level 0. Added `focused_block_mutable()` accessor to `UiState` and wired through `BuilderServices` → `ReactiveEngine`.
- Implemented `chain_ops(level)` in `value_fns/chain_ops.rs`: composition convenience that looks up `focus_chain[level]` and delegates to shared `ops_rows_for_uri()` helper. Both value fns registered in `build_shadow_interpreter` and added to `render_dsl.rs` widget_names for Rhai parser acceptance.
- Integrated mobile action bar into `block_profile.yaml`: root_layout `if_space(600, ...)` branch now wraps in nested columns/rows binding `collection: focus_chain()` at outer level and `collection: chain_ops(level)` at inner level, with icon/spacer/text row templates.
- Updated `shadow_builders/columns.rs` (raw fn) to mirror macro's collection precedence: reads `ba.args.get_rows("collection")` before upcast.
- Migrated `concat` from inline special case in `eval_to_interp` to registered `ValueFn` behind `CORE_VALUE_FN_LOOKUP` (replacing `EMPTY_VALUE_FN_LOOKUP`). Removed `legacy_concat` shim; `ValueFnBinding::invoke` now chains registered fns → core lookup.

**Challenges**

- `UiState.focused_block` is a signal; needed new `focused_block_mutable()` accessor to expose cloneable `Mutable` for reactive subscription in value fns.
- Mobile action bar YAML branch (`if_space(600, ...)`) never fired during PBT because viewport isn't set. Documented trigger pattern for next session; no fix applied yet.

**Decisions**

- Deferred full parent-walk in `focus_chain` (level > 0) to future iteration; current impl handles single-level focus for action bar composition. Sync parent lookup requires `UiState` schema change.
- Created handoff document (`docs/HANDOFF_VALUE_FN_PBT_INVARIANTS.md`) for `general_e2e_pbt.rs` integration: details 3 remaining invariants (arg variance, cache identity stability, no-flicker re-render), prerequisite `ProviderCache::get_or_create()` plumbing per fn, and `ReferenceState` focused_block tracking gap.
