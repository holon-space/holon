## Setting up code quality and duplication checks

The project needed automated code quality gates that could run both locally and in CI. Multiple crates across the monorepo weren't being linted consistently.

**What happened:**

- Added **deny.toml** with cargo-deny config (license audit, security advisories, dependency bans); required marking all 17 workspace crates with `publish = false` so they're recognized as private
- Added **.jscpd.json** for copy-paste detection with a 10% threshold, tuned to Rust only (Dart frontend had 40% duplication from generated code)
- Created four new justfile recipes: `fmt-check`, `deny`, `machete`, `duplication`, and `lint` (runs all)
- Updated **ci.yml** with new `rust-quality` job running in parallel:
  - cargo-deny for dependency/advisory checks
  - cargo-machete for unused dependency detection  
  - jscpd for code duplication
  - Broadened path filters to cover all Rust frontends and upgraded clippy scope

**Challenges:**

- **cargo-deny v0.19 config format**: v0.19 removed the `unlicensed` field; solved by adding `publish = false` to all workspace crates so `private.ignore = true` works correctly
- **High Dart duplication**: Flutter frontend had 40% duplication from generated code and multi-implementation patterns; focused jscpd on Rust only to catch real regressions (1.65% final result)
- **cargo-machete false positives**: workspace-hack crate generates noise but tool still serves its purpose

**Status:** All three tools passing locally and integrated into CI; developers can run `just lint` to check all quality gates before pushing.
