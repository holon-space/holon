## Fixing Property-Based Tests for EditViaDisplayTree and Related Transitions

**Context**: The `general_e2e_pbt.rs` property-based tests were panicking when executing `EditViaDisplayTree` and `TriggerSlashCommand` transitions due to attempting to render leaf blocks (text/headline) using `render_entity()` API, which requires query source children.

**What happened**:

- Diagnosed root cause: `EditViaDisplayTree` called `engine.blocks().render_entity(block_id)` on arbitrary text blocks, but `render_entity` API only works on blocks with query sources
- Identified that in production, leaf blocks are rendered via entity profiles through parent layout contexts, not via `render_entity` API
- Fixed preconditions in `state_machine.rs` to restrict `EditViaDisplayTree`, `EditViaViewModel`, and `TriggerSlashCommand` to blocks visible in the main region's focus tree (descendants of focus roots)
- Updated `sut.rs` `apply` handlers to query the block's data directly and create a `RenderContext` for rendering, rather than calling the API's `render_entity` method
- Reverted unnecessary changes to AppState creation and `current_view_model()` usage

**Challenges**:

- Initially attempted using `current_view_model()` from AppState, but this returns `None` before watches receive first snapshots, and returns empty data tables due to async GQL processing
- Realized the correct path: render leaf blocks using entity profiles with row data context, exactly like production does via parent layout render expressions
- Encountered invariant failures around source blocks in matview vs reference model — identified as pre-existing filtering mismatch

**Decisions**:

- Restrict block selection to focus-visible blocks rather than all text blocks in database
- Query block data directly and create render context instead of using async watch-based rendering paths

**Status**: `general_e2e_pbt_sql_only` passes; `Full` and `CrossExecutor` variants now hit invariant 10i failure related to source block filtering in matview — investigating whether this is pre-existing or introduced by focus tree filtering.
