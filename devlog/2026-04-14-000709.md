## Project Wiki Initialized with Tech Debt Tracking

A living wiki was generated for the Holon codebase using the Karpathy LLM Wiki pattern, documenting 23 pages of architecture, components, and patterns. During generation, structural inconsistencies were identified and captured as trackable tech debt.

**What happened:**

- Scanned codebase structure, read existing docs and git history
- Generated `overview.md` with architecture, tech stack, and directory layout
- Created 9 entity pages (services, crates, frontends) and 10 concept pages (auth flows, reactive patterns, CRDT, testing strategies)
- Built `index.md`, `log.md`, and `schema.md` to establish wiki conventions and operations
- Identified 8 architectural inconsistencies during code review and documented them in `tech-debt.md`
- Updated `CLAUDE.md` to point to wiki as the source of truth
- Modified INGEST procedure to review `tech-debt.md` on every wiki update

**Challenges:**

- **Tech debt discoverability** — Inconsistencies found during wiki generation risked being lost as untracked observations. Solved by creating `tech-debt.md` as a first-class wiki page with severity levels and fix directions, explicitly integrated into the INGEST contract so it gets reviewed and updated whenever code changes.

**Decisions:**

- **State-based, not history-based** — INGEST relies on reading actual source files rather than `git diff`, making the wiki resilient to squashing and non-linear VCS workflows
- **Tech debt as wiki maintenance** — Rather than external issue trackers, tech debt lives in the wiki alongside architecture docs, ensuring both evolve together

**Status:** Wiki is live at `wiki/index.md` with 23 pages and integrated tech debt tracking. Ready to INGEST on next significant code changes.
