## Fixed GPUI Build Against Nightly Rust Breakage

The GPUI project was failing to compile against the latest nightly due to upstream rustc intrinsic changes.

**What happened**

- Attempted to run `cargo run -p holon-gpui --features chrome-trace` on aarch64-apple-darwin with nightly toolchain
- Build failed with `E0425: cannot find function simd_fmin` and `simd_fmax` errors in `pathfinder_simd` 0.5.5
- Discovered servo/pathfinder#584 — rustc removed `simd_fmin`/`simd_fmax` intrinsics on nightly-2026-03-29, renaming them to `simd_fminimum`/`simd_fmaximum`
- Pinned `rust-toolchain.toml` to `nightly-2026-03-28` (last known-good date before the breakage)
- Verified build succeeds: 924 crates compile without errors
- Successfully launched the app with chrome-trace feature enabled

**Challenges**

- Initial diagnosis: needed to identify the exact rustc version cutoff where the intrinsic rename occurred — tested nightly-2026-03-30 and nightly-2026-03-28 to bracket the breakage
- Lock contention: previous cargo processes held locks; resolved by waiting for cleanup
- Log volume confusion: the compiled app produced 1.46M lines of TRACE-level output on startup, initially appearing as a "loop" but actually just verbose tracing from the Turso/holon runtime

**Status**

Build is fixed and app launches successfully. The 924-crate compilation takes ~2min due to full toolchain change rebuild. Runtime logging is very verbose at TRACE level but normal.
