## Fixed screenshot tool to correctly identify GPUI main window

The MCP server runs in the same process as the GPUI frontend, causing `capture_window_as_png` to match invisible auxiliary windows created by GPUI instead of the main application window.

**What happened**

- Identified root cause: `capture_window_as_png` (line 1895 in `@frontends/mcp/`) was matching the first window by PID, which enumerated GPUI's internal 500x500 auxiliary window before the main "Holon" window
- Initially explored fixing by selecting the largest window by area instead of first match
- Added debug logging to `/tmp/holon-screenshot-windows.txt` to inspect all windows for the process
- Discovered the auxiliary window has an empty title (`""`), while the main window is always titled `"Holon"` (set in `lib.rs:611`)
- Switched to matching by **PID + exact title `"Holon"`** instead of just PID, which uniquely identifies the correct window
- Removed debug logging code after validation

**Challenges**

- Initial assumption that area-based heuristic would be stable, but realized title matching is more robust — window resizing could break area-based logic
- Multiple "Holon"-containing windows exist in user's environment (Zed editor, VCS tool), so PID alone wasn't sufficient — needed the exact title filter

**Decisions**

- Use PID + exact window title instead of PID + area heuristic: simpler, more predictable, and doesn't break on window resizing

**Status**

Screenshot tool now correctly captures the main Holon window from both `holon-direct` and `holon-live` instances, showing all three panels (sidebar, main, right panel).
