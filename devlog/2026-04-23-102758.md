## Enhanced squash-frontends.sh with Selective Folder Support

The script needed flexibility to squash only specific folders instead of always auto-discovering from bookmarks, while maintaining backward compatibility.

**What happened:**
- Updated `scripts/squash-frontends.sh` to accept optional folder arguments
- Implemented three usage modes: auto-discover (default), specific folders, and `--dry-run` mode
- `--dry-run` can appear anywhere in the argument list and skips actual commits
- Tested with multiple folders (`specs` and `devlog`) to verify multi-squash workflow
- Verified that descendant commits are properly rebased after each squash operation

**Status:** Complete and tested. Script now supports `squash-frontends.sh [--dry-run] [folder1 folder2 ...]` with full backward compatibility.
