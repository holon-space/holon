## Drag-Drop Polling & Router Bootstrap Fixes

**Context:** The drag-drop PBT strategy was disabled due to missing infrastructure. The goal was to add a polling helper so `drop_entity` could wait for widgets to actually render before attempting the operation, unblocking gate-flip and end-to-end testing.

**What happened:**

- Added `drop_widget_timeout()` helper (mirrors `quiescence_config()`) with default 5s timeout for widget visibility polling in `ReactiveEngineDriver::drop_entity`.
- Discovered router was never initialized for DragDropBlock when no prior key chord had fired. Fixed by passing `root_block_id` to `drop_entity` and calling `ensure_block_watch(root_block_id)` on entry — mirrored `send_key_chord` pattern.
- Updated trait signature across all implementations (Direct, GPUI, Flutter) and SUT call site.
- Added `HeadlessInputRouter::diagnostic_snapshot()` to dump watches, block trees, and `(block_id, row_id)` tuples on timeout for debugging.
- Verified PBT now fires DragDropBlock (21 attempts) with cleaner error messages instead of silent failures.
- All affected crates compile clean: `holon-frontend`, `holon-integration-tests`, `holon-gpui`.

**Challenges:**

- **Widget binding gap**: Discovered draggable/drop_zone widgets rendered through `block_profile.yaml` never get `row_id` populated — root cause is structural widgets don't propagate entity id into their data map. This is also a **production bug** (GPUI's draggable.rs short-circuits on None row_id). Documented as open point #9 in handoff; requires interpreter-level fix.
- **Pre-existing PBT divergence**: SplitBlock strategy was already failing on main branch (unrelated to drag-drop). Disabled via `PBT_WEIGHT_SPLIT_BLOCK=0` to isolate drag-drop validation.

**Decisions:**

- **Eager router bootstrap**: Pass root_block_id to drop_entity rather than lazy-init, matching send_key_chord pattern for consistency.
- **Keep gate disabled**: Left `DRAG_DROP_ENABLED=false` with updated comment pointing to #9; the polling/bootstrap layer is solid but blocked on structural widget binding.

**Status:** Polling helper and router bootstrap implemented and tested. Gate remains disabled pending fix to widget row_id propagation in block rendering layer. All infrastructure changes documented in `drag_drop_wiring.md`.
