## Rigorous Code Review of Image Support Plan

**Context**: A plan to add image block support to the Holon org tool was reviewed as a senior dev doing a scathing code critique. The session involved reading the plan, auditing the actual codebase, and exposing fatal assumptions and missing pieces.

**What happened**:
- Identified that "already implemented" frontend code exists only in a conflicted Claude Code worktree (`feat-image-support`), not on main — a critical gap
- Discovered `BlockContent::Image` enum variant is architecturally inconsistent with how the codebase stores content metadata (flat fields vs. JSON properties bag)
- Found a silent data corruption vector in `loro_backend.rs:105` where `Some("image")` would fall through to text block fallback
- Exposed that org sync would destroy image blocks on round-trip because the parser (Slice 3) was deferred while writing (Slice 1) shipped immediately
- Uncovered missing implementations: `image()` render DSL function, shadow builder, GPUI builder all absent from main
- Identified PBT test generators don't create image blocks, leaving image rendering as a blind spot
- Flagged directory traversal vulnerability (no path jail) and `attachments/` lifecycle issues (git handling, hash collisions, directory creation)

**Challenges**:
- The plan treated worktree code as "done" when it was actually divergent and unmerged — required recalibrating scope
- Org round-trip data loss was the core blocker: writing without parsing creates a time bomb that detonates on file watcher or restart
- Abstraction mismatch meant refactoring `BlockContent::Image` out entirely and using `ContentType::Image` with path in `content` field instead

**Decisions**:
- Made org parser and renderer atomic (Step 3) — cannot ship write without read
- Simplified from enum variant to flat content type to match existing patterns
- Added explicit panic on unknown `content_type` values instead of silent fallthrough
- Deferred `attachments/` git strategy and hash collision handling to validation tasks

**Status**: Plan was comprehensively rewritten incorporating 8 critical hypotheses and their mitigations; architectural mismatch resolved and data loss risk eliminated.
