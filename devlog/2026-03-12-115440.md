## Architecting CDC-based LiveData flow from Turso to UI

Building a reactive dataflow system where change data capture (CDC) events from Turso automatically update a LiveData cache layer that feeds the UI.

**What happened**

- Established the architecture pattern: Turso database → CDC event stream → LiveData cache → UI layer
- LiveData acts as a smart cache that combines initial snapshot data with incremental CDC updates, eliminating need for full refetches
- Identified the key advantage: CDC stream provides only deltas, allowing the cache to stay fresh without network overhead
- Designed the dataflow to be unidirectional and reactive, with UI automatically reflecting changes as they arrive
- Discussed how this pattern handles both initial load (snapshot) and ongoing mutations (stream)

**Decisions**

- Used LiveData as the cache layer rather than raw state holders, leveraging its lifecycle awareness and automatic cleanup in Android contexts
- Kept CDC consumption at the cache layer rather than pushing raw events to UI, providing a clean separation of concerns

**Status**

Architecture validated and documented; ready for implementation of CDC consumer and LiveData integration logic.
