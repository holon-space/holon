## Refined Plan: Symmetric PBT Harness for TUI Frontend

## Context
The initial plan hallucinated infrastructure (nonexistent `RenderRegistry`, closed `MockInputDevice`, inaccessible `OffscreenBuffer`) and diverged from GPUI's test structure. A code audit found five blockers, but the solution was overcorrected — throwing out the symmetric architecture that made the design valuable. This revision restores the GPUI mirror while fixing only the genuine implementation gaps.

## What happened

- **Audited GPUI's PBT structure** (`gpui_ui_pbt.rs`) and mapped its five shared helpers, driver shape, and cross-thread handoff back onto TUI
- **Identified the minimal divergence points:** `dispatch_interaction` (event translation) and `run_capturing_app` (render loop) — everything else is GPUI-shaped
- **Fixed the five blockers without architecture thrashing:**
  - Introduced `TuiWidget::Tagged` wrapper variant + entity-aware `attach_entity` to make widgets carry entity ids through the render walk
  - Wrapped `AppMain` in a `CapturingApp` that hooks the public `RenderPipeline::compose_render_ops_into_ofs_buf` into an `Arc<RwLock<OffscreenBuffer>>` instead of trying to steal from private `GlobalData`
  - Swapped cell encoding from background → foreground so screenshot analysis works unchanged
  - Kept `MockInputDevice` closed-stream as-is; `dispatch_interaction` translates events to engine dispatch in v1, real input pipeline deferred to r3bl follow-up
- **Restored shared infrastructure:** identical `DebugServices` plumbing, `TuiUserDriver { tx, geometry, engine }` field-for-field match, same `PbtReadyResult` shape, `harness = false` test binary mirroring GPUI's `ctx_tx`/`window_ready_rx` handoff and optional MCP port
- **Updated plan with explicit tasks:** entity tracking, `CapturingApp` wrapper, foreground encoding validation, `dispatch_interaction` translation logic, screenshot threshold tuning

## Challenges

- **`OffscreenBuffer` privacy:** Solved by wrapping `AppMain` in `CapturingApp` and hooking the public compose API instead of accessing private fields
- **Entity IDs lost during render:** `TuiWidget::Tagged` variant ensures entities survive the walk, validated against current enum shape
- **Screenshot darkness:** Encoding foreground instead of background lets `analyze_screenshot_emptiness` detect non-empty cells; threshold choice (`brightness > 45`) requires TUI painter config verification

## Decisions

- **Keep `MockInputDevice` closed-stream; translate in `dispatch_interaction`.** Avoids forking r3bl in v1. Real input pump (async channel → event loop) and custom `TuiUserDriver` chord routing defer to a gated follow-up.
- **`CapturingApp` wraps `AppMain`, not `ReactiveEngine`.** Keeps the event loop running; the wrapper just reads composed buffers at render time instead of trying to share `GlobalData`.

## Status

Plan rewritten to restore GPUI symmetry while fixing the five implementation blockers. All shared helpers, driver shapes, and test structure now mirror GPUI. TUI-specific deltas confined to two function bodies. Ready for implementation phasing.
