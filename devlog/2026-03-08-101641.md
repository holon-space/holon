## Integrated gpui-component TitleBar into the GPUI frontend

The custom title bar implementation needed to be replaced with the proper `gpui-component::TitleBar` to get native macOS window chrome (traffic lights, dragging, etc.) without manual hacks.

**What happened**

- Replaced manual `build_title_bar()` DIV with `TitleBar::new()` from gpui-component, removing ~20 lines of custom layout code
- Updated `WindowOptions` in `cx.open_window()` to use `TitleBar::title_bar_options()`, which sets `appears_transparent: true` and configures proper `traffic_light_position` on macOS
- Added interactive buttons to the title bar to verify functionality: hamburger menu icon (left) and search/settings icons (right), styled as ghost buttons
- Verified the result compiles cleanly and runs without errors

**Decisions**

- Used `ghost()` button variant so icons blend seamlessly into the title bar rather than looking like standalone UI elements
- Kept button actions unwired for now; they're visual placeholders for future sidebar toggle, search, and settings functionality

**Status**

TitleBar is now rendering with native macOS traffic light buttons, 34px standard height, and interactive icon buttons. Ready for wiring up actual functionality.
