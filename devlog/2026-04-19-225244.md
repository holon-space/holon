## Mobile Operation Triggering with Dynamic Action Bars

Mobile devices lack keyboard shortcuts, requiring an alternative UI pattern to trigger operations. The goal was to implement a dynamic action bar (similar to LogSeq's approach) that displays context-sensitive operation buttons above the soft keyboard.

**What happened:**

- Identified need to create `ViewModel` and `ReactiveViewModel` wrappers for `OperationDescriptor`s to enable reactive UI binding
- Discussed architectural approach: extend `EntityProfile` system to support `OperationDescriptor` as a first-class entity type, allowing declarative UI configuration via YAML
- Planned integration with existing cursor position tracking to make the action bar context-aware — operations displayed should reflect the entity under cursor
- Considered two implementation paths: (1) dedicated `EntityProfile` YAML for `OperationDescriptor` type, or (2) inline operation declarations within existing entity profiles
- Discussed injection points for making operation viewmodels available to UI rendering layer without coupling to specific UI frameworks
- Explored how this pattern could support both mobile action bars and desktop context menus/pie-menus from the same operation definitions

**Decisions:**

- Use `EntityProfile` YAML as the single source of truth for operation metadata — this keeps the declarative pattern consistent and avoids special-casing operation handling
- Each `OperationDescriptor` needs explicit `EntityUri` and `EntityName` properties to integrate with the profile system
- Bind action bar visibility and content to cursor position via existing reactive systems rather than creating parallel tracking logic

**Status:**

Design phase complete; ready to implement `OperationDescriptor` ViewModel wrappers and extend `EntityProfile` YAML schema to support operations.
