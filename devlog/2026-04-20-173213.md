## Documented Claude Code Advisor Feature Reference

A comprehensive reference guide for the Claude Code advisor feature was created and added to persistent memory for future reference.

**What happened:**
- User asked clarifying questions about the advisor feature: model pairing constraints, token economics, and caching behavior
- Documented that valid executor/advisor pairs require advisor capability ≥ executor (Opus 4.7 is only supported advisor; Haiku/Sonnet/Opus 4.6 can be executors)
- Explained full conversation transcript is sent to advisor, including all tool results—no truncation
- Identified advisor input token cost as primary expense (~$5/M for Opus 4.7), with output caching as key mitigation
- Created `memory/claude_code_advisor_feature.md` capturing model pairings, token billing breakdown, ephemeral caching configuration, and cost control strategies
- Indexed new memory file in MEMORY.md for persistence

**Challenges:**
- **Cache invalidation risk**: Using `clear_thinking` with `keep` settings other than `"all"` shifts transcript structure each turn, busting the advisor cache despite ephemeral caching being enabled—noted as a caveat for users
- **Token tracking opacity**: Advisor usage appears in `usage.iterations[].type: "advisor_message"` rather than top-level usage fields, requiring separate monitoring

**Decisions:**
- Documented caching break-even at ~3 calls, making it non-essential for short tasks but critical for long agent loops
- Recommended explicit call budgeting and removal of advisor tool from available tools once limit reached, rather than relying on context window failures

**Status:** Advisor feature reference documented and indexed in memory for future sessions.
