## Handoff: `columns()` partition width calculation ‚Äî planning session

### Context

The Holon render system uses a shadow interpreter that produces a `ReactiveViewModel` tree. The `columns()` shadow builder partitions available screen width equally among its children so that `if_space()` and profile variant conditions inside each child see the correct slot width (container-query pattern). This equal partition is wrong in several cases where children have a known or deterministic size that is not "one equal share."

A `DrawerMode` enum (`Shrink` / `Overlay`) was just added. The goal of this session is to design the correct partition calculation.
