## Matview-on-Matview CDC Propagation Bug in Turso

**Context**: While fixing PBT timing issues in holon, discovered that CDC events don't propagate through chained matviews.

**What happened**:
- Created minimal reproducers testing matview CDC in various scenarios
- Confirmed basic matview CDC works correctly (test 1)
- Confirmed basic chained dependency CDC works (test 2)
- **Found the bug**: matview-on-matview CDC breaks (test 3)

**The bug**:
When a table is modified, the change correctly propagates to a matview depending on it. But if a *second* matview queries the first matview, changes don't propagate to the second matview's CDC stream, and the second matview's data stays stale.

**Minimal reproduction**:
1. Create `focus_roots` matview (joins `current_focus` matview with `block` table)
2. Create downstream matview `mv_region_watch` that queries `focus_roots`
3. Insert new row into `block` table
4. Result: `focus_roots` CDC fires correctly, but `mv_region_watch` CDC gets **zero events** and query data is stale

**Evidence** from test output:
- `focus_roots` matview: ✓ 1 CDC event, correct data
- `mv_region_watch` matview: ✗ 0 CDC events, stale data (`["b1"]` instead of `["b1", "b2"]`)

**Decision**: Create handoff to Turso repository rather than work around this in holon. The bug is in Turso's IVM CDC propagation through chained matviews; fixing it at the source is cleaner than adding workarounds.

**Status**: Reproducer confirmed and documented in `libsql_ivm_integration_tests.rs`. Ready for Turso handoff.
