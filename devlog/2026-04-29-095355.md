## Upgraded code-graph-mcp to embedding-enabled binary

The npm-distributed `code-graph-mcp` binary was built without the embedding model, limiting similarity searches to text-based FTS5 fallback. We needed the semantic embedding version to properly find similar code.

**What happened:**

- Checked available tools in code-graph MCP — found `semantic_code_search`, `get_call_graph`, `get_ast_node`, and `trace_http_chain`
- Discovered the CLI `similar` subcommand requires embeddings, which weren't built into the npm distribution
- Evaluated three paths: text-based search fallback, MCP tools (same limitations), or building from source with embeddings enabled
- Chose `cargo install --git` approach for simplicity — single command, default features include `embed-model`
- Provided installation and verification steps (`code-graph-mcp doctor`) plus PATH management to ensure Claude Code uses the new binary over the npm version

**Challenges:**

- npm package ships lighter build without embeddings — no control over that from the consumer side
- Candle embedding model adds ~100MB and requires background download of model weights on first run

**Decisions:**

- Went with `cargo install --git` over local build — avoids maintenance burden of manual rebuilds
- Kept `--locked` flag to respect Cargo.lock; mentioned dropping it if transitive deps cause issues

**Status:** Installation command provided. User can now run `cargo install --git https://github.com/sdsrss/code-graph-mcp.git --locked`, reconnect MCP, and use semantic similarity for `ReactiveViewModel` and other symbols.
