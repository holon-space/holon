## Investigating Race Condition in Holon Document Loading

**Context:** When relaunching Holon with the `Holon` document previously selected, the main panel displays only a subset of blocks from `index.org` instead of the full set. Navigating away and back restores all blocks, suggesting a timing/initialization issue.

**What happened:**
- User reported missing blocks on app startup when `Holon` document is in focus
- Set up `holon-live` MCP server to inspect live application state
- Confirmed the app was running with 266 rows in the main panel
- Began investigating the query driving the main panel to identify the discrepancy between expected and actual block counts
- Session was interrupted before completing root cause analysis

**Challenges:**
- Initial `holon-live` connection failed — server wasn't running; required user to start the app first
- Needed to determine actual vs. expected block count from `index.org` to quantify the issue

**Status:** Investigation in progress — race condition confirmed through user behavior but root cause in the main panel query not yet identified. Recommend checking: (1) block query execution timing relative to document selection, (2) initialization order in sidebar vs. main panel components, and (3) whether query filtering is incomplete on initial load.
