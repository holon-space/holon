## Fixed Render DSL Syntax Error and Log Truncation in Turso Exception Handling

A Turso CDC row was causing a panic due to invalid render DSL syntax, but the error message made it look like Turso was truncating data when it wasn't.

**What happened**

- Inspected `/tmp/holon-gpui-turso-bug.log` and found a panic: `Syntax error: Expecting ',' to separate the arguments to function call 'state_toggle'`
- Discovered the error message was truncating DSL expressions to 100 characters (line 50 of the logging code), making `row(state_toggle(col("task_state"), states: col("todo_states")), ...)` appear as `...editable_text(col("conte`
- Identified the root cause: `states: col("todo_states")` uses bare `key: value` named argument syntax, which Rhai doesn't support—it requires `#{states: col("todo_states")}` map syntax
- Fixed the error message to log the full DSL expression instead of truncating it
- Updated the `default-block-profiles` entity profile to use correct Rhai map syntax: `state_toggle(col("task_state"), #{states: col("todo_states")})`
- Merged the improved entity profile structure from `index.org` into `__default__.org`, including the `todo_states` computed field and `query_block` variant, while stripping example child blocks

**Decisions**

- Chose to preserve full expression logging for better debugging rather than truncating—this prevents false conclusions about data corruption
- Used `#{...}` map syntax to pass named arguments to `state_toggle`, following Rhai conventions where trailing maps are merged into named args

**Status**

Log truncation fixed, render DSL syntax corrected, and entity profiles unified in `__default__.org`.
