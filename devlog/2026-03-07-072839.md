## Refactored GPUI render interpreter to use shared Builder<W> trait

The GPUI frontend's custom render dispatch was replaced with the generic `RenderInterpreter<W>` infrastructure from holon-frontend, eliminating duplicate builder logic across frontends.

- Read all 28 GPUI builder files and the current dispatcher in `builders/mod.rs`, `interpreter.rs`, and `main.rs`
- Created `create_interpreter() -> RenderInterpreter<Div>` in `builders/mod.rs` that registers all builders by name (color, text, clickable, button, etc.)
- Converted all 28 builder function signatures from `fn build(args: &ResolvedArgs, ctx: &RenderContext)` to `fn build(ba: BA<'_>)` where `BA = BuilderArgs<'_, Div>`
- Updated all 28 builders: replaced `args.` → `ba.args.`, `ctx.` → `ba.ctx.`, and `interpret(expr, ctx)` calls → `(ba.interpret)(expr, ctx)` for recursion
- Replaced `live_query` builder's hand-written PRQL compilation with `shared_live_query_build(&ba)` and matched on `Ok/Err`
- Kept `block_ref` and `render_block` using `BlockRenderCache` (GPUI's async approach) rather than `shared_block_ref_build` (synchronous thread::scope)
- Updated `main.rs` to call `builders::create_interpreter()` and use `interpreter.supported_widgets()` instead of a hardcoded widget list

**Status**: Compilation clean (zero errors, zero warnings from holon-gpui). All builder dispatch now flows through the shared `RenderInterpreter<Div>` registry.
