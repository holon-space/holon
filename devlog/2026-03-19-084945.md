## Evaluated Macro Dependency: `zyn` and `darling` for `holon-macros`

Investigated whether adopting the `zyn` procedural macro library or the `darling` attribute parser would improve the `holon-macros` crate.

- Reviewed `zyn` blog post (returned 403; used web search instead) and assessed its template DSL benefits against current macro structure
- Examined all macro files: `entity.rs` (~320 lines), `operations_trait.rs` (~1300 lines), `widget_builder.rs`, `builder_registry.rs`, and `attr_parser.rs` (~270 lines)
- Analyzed `attr_parser.rs` to evaluate whether `darling`'s typed attribute extractors would replace the custom `AttrArgs` framework and attribute structs like `EntityAttribute` and `ReferenceAttribute`
- Traced the complexity hotspots: code generation logic in `operations_trait.rs` (dispatch cases, type-based branching, return type handling) vs. attribute parsing

**Decisions**

- Rejected `zyn` adoption: macros lack deeply-nested quote-in-quote patterns that zyn's template DSL targets, and the v0.2.0 maturity level doesn't justify adding compile-time overhead to a proc-macro crate
- Rejected `darling` adoption: attribute parsing is already solved cleanly with ~90 lines of custom code; the real complexity (80%+ of work) lies in code generation logic that `darling` cannot address, making it pure churn for negligible lines saved

**Status**: No dependency changes recommended. Current architecture is clean and well-suited to the problem domain.
