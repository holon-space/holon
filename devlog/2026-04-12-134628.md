## Fixed cycle detection errors in shadow index range adjustment

Arrow-key navigation was causing `bubble_input: cycle detected after 2205 steps` errors in the shadow index. The root cause was stale range metadata on outer blocks when inner blocks were patched.

**What happened:**

- Identified that `patch_shadow_block` was being called on every `ReactiveShell::reconcile_children`
- Found the bug in `crates/holon-frontend/src/shadow_index.rs:169-175`: the range-adjustment loop only updated `block_ranges` whose `start >= old_range.end`, ignoring ranges that *contained* the patched range
- Traced the failure mode: when inner block B grew from `[15..20]` to `[15..26]`, outer block A's range `[10..30]` stayed stale even though its subtree now extended to 36
- On the next patch of A, the stale range caused `splice()` to operate on wrong indices, corrupting parent pointers and creating cycles in the parent chain
- Verified this matched `/tmp/holon.log` evidence: repeated `patch_block: stale range, skipping` warnings with `range_end > nodes_len`

**Solution:**

- Extended the range-adjustment loop to handle containing ranges: when a stored range strictly contains the patched range, shift only its `end` by `delta`
- Added regression test `patch_inner_block_updates_outer_range_end` that reproduces the bug and verifies no cycles after follow-up patches
- Test failed before the fix (outer-range mismatch: `left: 7, right: 9`), passes after

**Status:** Cycle detection errors resolved; all 5 shadow_index tests passing.
