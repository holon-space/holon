## Fixed Screenshot Tool to Find Correct GPUI Window

The MCP screenshot tool was capturing an invisible 500x500 auxiliary window created by GPUI instead of the main "Holon" application window. The issue occurred because the MCP server runs in the same process as the GPUI frontend, causing PID-based window lookup to match multiple windows.

**What happened:**

- Identified that `capture_window_as_png()` used `.find()` on windows filtered by PID, returning the first match — which was GPUI's internal auxiliary window
- Added debug logging to dump all windows for the process to `/tmp/holon-screenshot-windows.txt`, revealing two windows: an untitled 500x500 invisible window and the actual "Holon" window
- Evaluated three approaches: title-based matching (rejected due to multiple Holon windows open in other apps), largest-area heuristic (initial fix), and PID + window title matching (final solution)
- Switched the lookup from PID-only to **PID + exact title "Holon"** match, uniquely identifying the main window
- Removed debug dump code after validating the fix worked on both `holon-direct` and `holon-live` instances

**Decisions:**

- Used PID + title matching instead of title-only matching to avoid collisions with other applications' "Holon" windows (e.g., Zed editor, VCS tools)
- Hardcoded the title "Holon" (set in `lib.rs:611`) as a stable discriminator rather than relying on window size heuristics that could be affected by user resizing

**Status:** Screenshot tool now correctly captures the main Holon application window from the MCP server, displaying all three panels (left sidebar, main panel, right sidebar).
