## Added ply-engine frontend to @frontends/

A new immediate-mode UI frontend using ply-engine (built on macroquad) was integrated into the holon renderer system, following the established interpreter + builder pattern.

**What happened:**

- Studied existing GPUI/Blinc frontends and the `RenderInterpreter` + `builder_registry` macro pattern to understand the plugin architecture
- Analyzed ply-engine's immediate-mode API (closure-based widget composition via `Ui` context) and macroquad integration
- Created 39 files at `frontends/ply/`: `Cargo.toml`, `src/main.rs`, `state.rs`, `cdc.rs`, `render/mod.rs`, `render/interpreter.rs`, and 27 builder modules matching GPUI's feature set
- Defined `PlyWidget = Box<dyn Fn(&mut Ui) + Send + Sync>` to represent widgets as closures that mutate UI state during render
- Implemented the `RenderInterpreter` to walk `RenderExpr` trees and dispatch to builder closures
- Fixed ply API usage throughout: `LayoutDirection::TopToBottom`, `gap(u16)`, `font_size(u16)`, `Sizing::Fixed()`, `AlignY::CenterY`, `Padding::new()`
- Added `extern crate macroquad_ply as macroquad` to satisfy the `macroquad_ply::main` macro's internal crate name expectations
- Verified clean compilation with no errors or warnings

**Challenges:**

- **Immediate-mode vs. retained-mode mismatch**: Ply doesn't return widget trees; it mutates a `Ui` context. Solved by representing widgets as closures instead of `Div` expressions.
- **Macro crate naming**: The `macroquad_ply::main` attribute macro internally refers to `macroquad` by name, but the crate is imported as `macroquad_ply`. Solved with `extern crate macroquad_ply as macroquad` alias.
- **API discovery**: ply-engine's docs were sparse; learned correct signatures by iterating: `gap()` not `gaps()`, `font_size(u16)` not `f32`, `Sizing::Fixed()` for dimensions, etc.

**Status:** Compiles cleanly; frontend scaffolding complete. Text measurement, `live_query` interactivity, click/drag handling, and text editing remain stubbed. Handoff document written to `frontends/ply/HANDOFF.md` with prioritized next steps and API references for fresh session.
